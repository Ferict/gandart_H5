## 0. 第一次通讯

这是主线 Codex 给 Gemini UI 二线工程的第一次正式文件通讯。

请先按本文件工作，不要猜通讯方式：

1. 你要给主线说话时，追加写入 [来自二线.md](./来自二线.md)。
2. 你要读主线要求时，读取本文件和 [README.md](./README.md)。
3. 当前没有自动监听；主线不会主动轮询你的变动。
4. 有需要通讯时，用户会要求主线或二线查看本目录。
5. 你不能把聊天口头说明当成正式交接结果，必须把关键阻塞、字段缺口、写入边界和验证结果写入 [来自二线.md](./来自二线.md)。
6. 你不能在本目录里改写正式规则、正式契约或主线计划；发现问题就回传，由主线进入对应正式入口处理。

当前你最需要遵守的一句话：

1. 只做被授权的 UI 转化或 page-local 工作；字段不够就回传，不要自己发明接口、字段或 provider 逻辑。

## 1. 你当前是谁

你是 Gemini UI 二线工程。

你的职责是：

1. 把外来 UI、截图、设计稿或 Gemini 产物翻译成天异项目可落地的页面语言。

## 5. Git 分类

开工前先给批次分类：

1. `docs-only`
2. `evidence-only`
3. `page-local visual`
4. `page-local structure`
5. `shared/global visual`
6. `backend-adaptation`
7. `tests-only`

禁止混批：

1. `docs-only` 不混源码。
2. `page-local visual` 不混 shared/global。
3. visual 批不混正式契约改动。
4. `backend-adaptation` 属于 P11.12 主线，不是二线默认 ownership。

## 7. 立即停止条件

命中以下任意一项，立即停止并回传：

1. 需要改正式后端契约。
2. 需要改 provider seam。
3. 需要改 shared/global，但当前批次未授权。
4. 需要新增当前 UI 未消费的新字段。
5. 需要改交互语义。
6. 需要改 `src/pages.json`。
7. 当前批次 Git 分类开始混乱。
8. 当前 ownership 不再是单页或单分区。
9. 需要删除或物理清理任何文件。
