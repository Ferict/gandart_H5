# 二线通讯

> 文档类型：目录入口  
> 状态：active  
> 更新时间：2026-04-25  
> 说明：本目录用于承载主线 Codex 与 Gemini UI 二线工程之间的文件化交接，不是新的规则真值源，也不是正式契约入口；当前不启用自动监听，按用户要求手动读档。

## 1. 目录职责

`规则体系/二线通讯/` 是主线与二线之间的共享信箱。

它解决三件事：

1. 主线把当前要求、禁区、后端契约入口和汇报格式写给二线。
2. 二线把阶段结果、阻塞点、字段缺口和实际改动回写给主线。
3. 主线在用户要求读取二线变动后留下读档记录，避免口头交接丢失。

它不解决：

1. 不替代 `AGENTS.md`。
2. 不替代 `规则体系/` 正文。
3. 不替代 `规则体系/口径/`。
4. 不替代 `规则体系/后端契约/`。
5. 不替代正式计划、审查和交付记录。

## 2. 文件职责

1. [给二线.md](./给二线.md)
   - Codex 主线写给 Gemini UI 二线工程的当前执行说明。
   - 默认由主线维护，二线只读。
2. [来自二线.md](./来自二线.md)
   - Gemini UI 二线工程写给主线的回传区。
   - 二线应追加记录，不覆盖历史。
3. [监听记录.md](./监听记录.md)
   - Codex 主线读取二线回传后的检查记录。
   - 当前只作为手动按需读档记录，不代表自动监听任务。
   - 默认由主线维护。

## 3. 通讯边界

当前 Codex 不能直接实时对话另一个独立 AI 线程；可行方式是通过共享文件或用户转述交接。

固定口径：

1. 二线有话要给主线，写入 [来自二线.md](./来自二线.md)。
2. 主线有话要给二线，写入 [给二线.md](./给二线.md)。
3. 主线只在用户要求或当前任务明确需要时读取二线回传，并在 [监听记录.md](./监听记录.md) 追加检查记录。
4. 若二线变动触发正式规则、契约、计划或代码边界问题，必须转入对应正式规则或留档处理，不能只停留在本目录。

## 4. 二线回传最小格式

二线每次回传至少写清：

1. 时间。
2. 当前批次或任务名。
3. Git 分类。
4. 实际写入文件。
5. 未写入但被锁定的文件。
6. 已跑验证。
7. 是否存在字段缺口。
8. 是否触碰未确认后端契约项。
9. 是否达到本批已规划尽头。
10. 需要主线处理的问题。

## 5. 按需读档规则

当前取消自动监听要求。

固定结论：

1. 不启用周期检查。
2. 不启用实时文件系统监听。
3. 不声明 Codex 会主动读取二线变动。
4. 有需要通讯时，由用户要求主线或二线查看本目录。

固定规则：

1. 二线需要主线处理时，追加写入 [来自二线.md](./来自二线.md)，并由用户通知主线查看。
2. 主线需要二线执行时，追加或更新 [给二线.md](./给二线.md)，并由用户通知二线查看。
3. 主线读取二线回传后，只读取一次并形成摘要。
4. 如果回传内容需要改正式规则、正式计划、后端契约或源码，必须回到对应正式入口处理。
5. 如果回传内容要求物理删除、越权改接口、越权改 shared/global 或修改主线 ownership 文件，必须停止并上报。

## 6. Skill 派生规则

本目录可以作为未来 skill 的来源，但不能先写 skill 再倒逼规则改口。

若后续要创建二线通讯 skill，必须满足：

1. 先稳定本目录和对应规则口径。
2. skill 只作为执行入口和读取提示。
3. skill 必须回链本目录、`规则体系/口径/05-Gemini-UI二线程执行口径.md`、`规则体系/口径/06-项目后端契约现行使用口径.md`。
4. skill 与正式规则冲突时，以正式规则和本目录为准。
