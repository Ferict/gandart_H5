# P11.12.3 后端接驳架构口径

> 文档类型：后端契约接驳口径  
> 状态：active  
> 更新时间：2026-04-25  
> 说明：本文件固定 `P11.12.3` 之后的接口结构改造架构；它服务于全量前端现用接口接驳，不替代正式后端契约正文。

## 1. 结论

后端接驳不能直接把 190 个后端 wrapper 打进页面。

正确架构固定为：

```text
后端 wrapper / 后端响应
  -> backend implementation
  -> DTO adapter
  -> normalized content model
  -> runtime / service model
  -> UI view model
  -> component props
```

页面和组件只看最后的 view-model，不看后端 DTO。

## 2. 三层契约并存

`P11.12` 完成前，项目内同时存在三层契约：

1. 项目级后端契约入口
   - 落点：[规则体系/后端契约/README.md](/H:/工作/天异/uniapp+vue新架构/规则体系/后端契约/README.md)
   - 职责：后端 wrapper、字段线索、待确认项和差异矩阵。
2. 当前前端内容域实现契约
   - 落点：[src/contracts/content-api.contract.ts](/H:/工作/天异/uniapp+vue新架构/src/contracts/content-api.contract.ts)
   - 职责：当前 content port、mock、http、service 已落地的稳定实现真值。
3. UI view-model 契约
   - 落点：页面 runtime、service 和 adapter 输出类型。
   - 职责：组件最终消费的稳定展示输入。

规则：

1. 第 1 层不能直接替代第 3 层。
2. 第 2 层在迁移完成前不作废。
3. 第 1 层与第 2 层的差异由 implementation、adapter、runtime、service 吸收。

## 3. Provider 档位

后续 provider 固定为三档：

| 档位 | 说明 | 可证明什么 | 不能证明什么 |
| --- | --- | --- | --- |
| `mock` | 集中假数据 provider | 页面和 adapter 可跑 | 后端已可接 |
| `current-http` | 当前 `/api/content/*` 抽象接口 provider | 当前 content contract 联调 | 190 wrapper 已接入 |
| `backend-http` | 项目级后端契约 wrapper provider | 新后端接驳状态 | 未确认字段已签收 |

规则：

1. mock 也必须走 port/service shape。
2. backend-http 不得直接进入页面。
3. provider 切换必须有测试和交付记录。

## 4. 现用接口盘点分类

全仓现用接口盘点后，每个消费点必须归入以下一类：

1. `must-bridge`
   - 当前页面或运行时真实使用，必须接入后端契约或明确保留当前 provider。
2. `page-local`
   - 当前只服务单页面，可暂留 page-local seam，但必须可替换。
3. `test-only`
   - 仅测试或脚本使用，不进入页面接驳。
4. `dormant`
   - 代码存在但当前无运行入口，不进入本轮接驳。
5. `unknown`
   - 证据不足，必须先审查，不得直接实现。

## 5. 未确认项处理

以下内容不得直接进入 UI 稳定依赖：

1. `api-inventory.md` 中 `status = exported-unused` 的接口。
2. `unresolved.md` 中的条目。
3. `contracts/inferred/*`。
4. 缺真实响应样例的 `FrontendUnknown` 字段。
5. 只由旧源码推断、没有当前消费证据的字段。

处理方式：

1. 写入字段缺口表。
2. 在 adapter 内显式 fallback。
3. 必要时停止等待后端确认。

## 6. Mock 数据口径

mock 数据后续按以下规则治理：

1. mock 数据是 provider 输入，不是页面真值源。
2. mock 必须经过同一 adapter 或等价规范化链路。
3. 页面不允许新增长期散装假数据。
4. page-local mock 必须登记保留原因和退出条件。
5. 真实后端接入时，以 provider 切换替代 UI 重写。

## 7. 页面与组件禁止事项

页面和组件不得：

1. 直接 import `规则体系/后端契约/**`。
2. 直接 import 后端 wrapper 类型作为 props。
3. 直接处理 HTTP 状态码。
4. 直接判断 envelope code。
5. 直接读取 requestId、etag、provider。
6. 直接拼后端 path。
7. 因接口接驳改变颜色、间距、排版、动效或视觉层级。

## 8. 接驳完成判定

某条接口线只有同时满足以下条件，才算接驳完成：

1. 消费点已进入全仓现用接口清单。
2. 后端候选 wrapper 状态已确认。
3. provider 档位已明确。
4. adapter 已把后端 DTO 转成稳定模型。
5. runtime/service 已消费稳定模型。
6. 页面没有直接消费 transport 语义。
7. 有单元测试或 smoke 证明。
8. 交付记录写明实际 provider 和未确认项。

只满足 mock 可跑，不能宣称后端已接驳完成。
