> 说明：本文件记录旧前端字段消费位置和接入风险，用于 P11.12 adapter 字段候选判断。

本文件回答“每个字段在旧前端哪里用、对新前端接入有什么要求”。它不是后端 DTO 注释，字段业务含义只按源码证据表达，不把猜测写成事实。

## Reading Rules

- `observed usage` 来自旧前端调用点、响应读取点、store/mixin/component 消费点。
- `backend support expectation` 只说明新前端复刻旧功能时后端需要继续支持什么字段，不声明源码没有证明的业务定义。
- `Level A/B/C` 仍以 `confidence.md` 为准；Level C 字段不能当 confirmed transport 字段。
- runtime derived 字段单独列出，不能混入正式后端响应结构。
- template/组件 prop 的展示字段在 `frontend-consumption-index.md`；只有能反查到 API dataflow 的字段才进入本文件的 transport field usage。
