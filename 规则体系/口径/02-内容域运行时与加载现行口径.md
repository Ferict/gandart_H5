## 2. 适用范围

当前正式适用：

1. 首页主面板
2. 活动页主面板
3. 个人中心主面板
4. 内容域 provider 默认行为
5. 内容域列表结果加载链与结果窗口链
6. 详情页当前刷新来源说明

当前不扩展：

1. 下游详情页 / 动作页的复杂结果链演出
2. 搜索框展开 / 收起 reveal
3. 排序弹层等局部浮层的独立动画口径
4. 非三主面板页面的全局加载动画扩散

## 3. 当前有效口径

### 3.1 当前 provider 与联调现实

1. 当前默认 provider 仍为 `mock`。
2. `http` provider 只在显式配置后启用。
3. 当前正式后端分页契约仍按 `page + pageSize` 理解。
4. 任何本地窗口大小、缓冲大小、显示批量口径都只属于前端运行时，不属于后端契约。

### 3.2 当前刷新与激活链

1. 三主面板继续分三层：
   - 首次进入
   - 后台静默更新
   - 激活校验 / 激活应用
   - 手动下拉刷新
2. 当前可见页在后台轮询命中新数据时仍只标脏，不当面自动换值。
3. 激活校验命中更新后，优先应用后台已缓存的新数据；无缓存时再做轻量拉取。
4. 手动下拉刷新仍是唯一允许明显刷新反馈的入口。
5. 详情页当前只接 `page-open + pull-refresh`，未接入三主面板的更高层更新链。

### 3.3 当前结果加载与动画基线

1. 当前视觉基线固定为：
   - 图片：淡入
   - 卡片：淡入淡出
2. 当前明确不回退到：
   - 图片扫描线
   - 卡片滑入滑出
3. 当前继续保留两条更新链：
   - 模块轻量更新
   - 列表结果加载
4. 列表结果区当前继续按三层模型实现：
   - 结果集
   - 结果窗口
   - 挂载窗口
5. retained 老卡默认只 patch；当前只有 `manual-refresh` 允许 retained replay。
6. `added / removed / retained` 的 keyed diff、placeholder gate、mounted window、overlay 离场继续沿当前结果链实现。

### 3.4 当前图片、footer 与占位口径

1. 当前图片加载占位与分页 loading、首屏 loading 继续分层。
2. 当前 placeholder 集合只做内部状态追踪，对用户不可见，不渲染可见骨架。
3. 当前 footer 继续常驻，`loading / 分页失败 / endline` 分层表达。
4. 当前只允许数据轻预取，不保留隐藏图片预加载主链。

### 3.5 当前分页 loading 与自动重试口径

1. 分页 loading 动画的出现条件等同于分页加载行为的触发条件；loading 不是装饰态，也不是“可能还有更多”的占位态。
2. 任一列表分页 loading 可见时，必须存在一条当前有效分页加载链路。
3. 当前有效分页加载链路只允许处于以下状态时显示 loading：
   - `requesting`
   - `retry-waiting`
   - `retry-requesting`
   - `no-progress-retry-waiting`
   - `no-progress-retry-requesting`
   - `committing`
4. 如果当前不存在分页加载行为、自动重试等待、自动重试请求或分页结果写入行为，分页 loading 不得出现。
5. `hasMore = true` 不是分页 loading 出现条件；`hasMore = true` 只是允许触发分页加载行为的前置条件。
6. 分页加载行为触发条件固定为：`hasMore = true + 视口到达触发区 + 未在请求中 + 未在过渡中 + 未达到自动重试停止条件`。
7. 上述触发条件成立时，必须立即进入分页加载行为并显示分页 loading。
8. 上述触发条件不成立时，不得仅因为 `hasMore = true` 显示分页 loading。
9. 同一列表同一时间只允许一条分页加载链路，执行 `singleFlight = true`。
10. 每条分页加载链路必须绑定当前 query signature、page、pageSize、sort、category、panel key 和请求版本，执行 `requestVersionGuard = true`。
11. 分类、排序、搜索、用户域、页面面板切换时，旧分页链路立即失效，执行 `cancelOnQueryChange = true`。
12. 当前面板失活时，分页链路停止，执行 `cancelOnPanelInactive = true`。
13. 视口距离当前已显示列表尾部小于等于 `8` 行时触发分页加载行为，执行 `triggerRemainingRowsThreshold = 8`。
14. 同一列表分页触发冷却时间为 `300ms`，执行 `loadMoreCooldownMs = 300`。
15. 单次分页请求超时时间为 `10000ms`，执行 `requestTimeoutMs = 10000`；无法中断底层请求时，必须用请求版本忽略超时后才返回的旧结果。
16. 分页请求失败时进入自动重试链路，不得无限保持普通 loading。
17. 同一分页触发链路最多自动重试 `3` 次，不包含首次请求，执行 `autoRetryMaxAttempts = 3`。
18. 自动重试等待间隔固定为 `[800, 1600, 3200]ms`，执行 `autoRetryDelaysMs = [800, 1600, 3200]`。
19. 自动重试等待和自动重试请求都属于加载行为，因此期间允许显示分页 loading。
20. 自动重试耗尽后仍未成功加载新增内容，必须结束分页 loading 并转为分页错误状态。
21. 分页成功但没有新增内容时，必须进入无新增判定链路，不得继续保持普通 loading。
22. 无新增包括：空页、重复页、`304 / notModified`、去重后 `appendedItems.length === 0`。
23. 无新增链路最多自动重试 `2` 次，执行 `noProgressMaxAttempts = 2`。
24. 无新增重试等待间隔固定为 `[800, 1600]ms`，执行 `noProgressRetryDelaysMs = [800, 1600]`。
25. 无新增停止策略固定为 `endline-if-page-advanced-or-total-reached-else-error`：
   - 如果页码已经推进，或当前数量已达到 `total`，判定为 `endline`。
   - 如果页码没有推进、数量也没达到 `total`，但连续无新增重试耗尽，判定为 `error`。
   - 不允许无新增后无限保持 loading。
26. 网络错误、接口错误、解析错误、超时错误的停止策略固定为 `error-after-auto-retry-exhausted`。
27. 手动“重试”只在 `error` 状态出现。
28. 用户点击手动“重试”后，必须开启新的分页加载链路并重置自动重试额度，执行 `manualRetryPolicy = reset-auto-retry-budget`。
29. footer 可以常驻，但 footer 常驻不等于 loading 常驻。
30. footer 状态只允许由当前有效分页加载链路决定：
   - 当前有效分页加载链路存在：`loading`。
   - 当前有效分页加载链路失败且自动重试已耗尽：`error`。
   - 当前列表确认没有更多：`endline`。
31. 已显示列表数量大于等于 `1` 时 footer 可以常驻，执行 `minVisibleItemsBeforeFooter = 1`。
32. 如果 footer 常驻但当前没有有效分页加载链路、没有错误、没有 endline，不得显示 loading；一旦满足分页触发条件，应立即创建分页加载链路。
33. `.home-list-loading-footer.is-loading` 可见时，必须能在运行时状态中追踪到一条当前有效分页加载链路，且链路状态属于本节第 3 条。
34. 首屏 loading、分页 loading、图片 loading 三者互不替代：
   - 首屏 loading 只绑定首屏数据请求。
   - 分页 loading 只绑定分页请求、分页自动重试、分页结果写入。
   - 图片 loading 只绑定图片资源请求。
35. 不允许用图片加载、卡片进入动画、占位渲染、`hasMore = true` 或 footer 常驻证明分页 loading 正在加载。
36. 用户看到分页 loading 时，系统语义固定为：当前列表正在尝试获得更多内容。

### 3.6 当前三主面板现实差异

1. `home / profile` 当前已进入更深的结果链与 result-window 拆分阶段。
2. `activity` 当前保留为相对稳定的参考实现，不为了对称继续强拆深层时序。
3. 后续若继续改 `home / profile` 深层 `enter / replay`，应先改本口径，再实施。

## 4. 当前明确不扩展范围

1. 不把当前 provider 默认值写回正式接口正文。
2. 不把当前列表结果链现实写成长期不变规则。
3. 不把当前三主面板加载动画直接扩散到详情页或非主面板页面。
4. 不把当前窗口大小、缓冲行数、局部 runtime 参数写成后端契约。

## 5. 何时必须同步更新本口径

命中以下任一情况时，必须同步更新本文件：

1. 默认 provider 或 `http` 联调入口发生变化。
2. 三主面板刷新来源、激活更新链、手动刷新语义发生变化。
3. 结果加载动画基线发生变化。
4. 列表结果链的 retained / added / removed / replay 语义发生变化。
5. 详情页刷新来源不再只是 `page-open + pull-refresh`。
