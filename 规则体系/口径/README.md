# 现行口径

> 文档类型：目录入口  
> 状态：active  
> 更新时间：2026-04-13  
> 说明：本目录用于承载当前仍有效的前端执行口径、当前实现基线和当前明确不扩展范围；它是“现状真值源”，不是“长期规则正文”。

## 1. 目录定位

`规则体系/口径/` 是新体系中的现行口径层。

它负责回答：

1. 当前实现基线是什么。
2. 当前哪些口径仍然有效。
3. 当前明确不扩展到哪里。
4. 当实现发生变化时，应该同步更新哪份口径文。

它不负责：

1. 代替 `AGENTS.md` 或 `规则体系/` 充当长期规则正文。
2. 承担审查记录、交付记录或历史证据保存。
3. 回读旧审批包文档作为现行真值源。

## 2. 三层关系

当前正式分层固定为：

1. `AGENTS.md + 规则体系/*.md`
   - 稳定规则、长期边界、禁止事项、验收要求
2. `规则体系/口径/*.md`
   - 当前实现基线、当前有效口径、当前不扩展范围
3. `留档/*`
   - 审查、交付、即时思路、历史记录

规则：

1. 口径文必须回链对应上位规则。
2. 规则正文不得继续混写“当前已知问题”“当前默认值”“当前仍只支持”等现状内容。
3. 留档可以引用口径文，但不替代口径文。

## 4. 更新规则

以下情况命中时，必须同步更新对应口径文：

1. 当前默认 provider、当前刷新来源、当前分页现实口径发生变化。
2. 当前页面壳与 page-local 实现边界发生变化。
3. 当前明确不扩展范围被突破。
4. 旧审批包中的现行口径被重新裁决并写入新体系。

## 5. 禁止事项

1. 不允许把旧审批包口径文继续当作现行真值源。
2. 不允许把当前实现基线继续散写回 `规则体系/*.md` 正文。
3. 不允许把现行口径写进 `留档/` 代替正式口径层。
