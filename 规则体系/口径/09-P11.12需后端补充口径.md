## 1. 职责边界

本文件负责承载：

1. 当前前端需要，但后端字段、wrapper、响应样例或业务语义尚未确认的事项。
2. 需要后端确认字段来源、格式、枚举、单位或 fallback 的事项。
3. 需要产品、后端和前端开会拍板的展示决策。

本文件不做：

1. 不把待补项写成后端已支持。
2. 不直接决定 UI 怎么展示。
3. 不承载真实 API path、method、DTO 或联调说明。

## 3. 待定项等级

1. `必须确认`
   - 没有确认就会影响当前 UI 基础接驳的字段，例如 G1 到 G6。
2. `会议待定`
   - 需要产品、后端和前端共同拍板的展示模块，例如 G7、G8。
3. `候选补充`
   - 可能影响后续功能完整度，但不阻塞当前首批接驳，例如 G9。

## 4. 后端补充格式要求

后端或后端契约补充时，每个事项至少需要给出：

1. wrapper 或 endpoint。
2. 请求参数。
3. 响应字段名。
4. 字段用途。
5. 单位或格式。
6. 缺失或为空时的返回方式。
7. 真实响应样例。

## 5. 前端处理规则

1. 待补项未确认前，adapter 可以保留 mock fallback，但必须标注为待确认。
2. 会议待定项未确认前，不进入稳定 view-model。
3. 任何待补项都不得直接写进页面模板。
4. 二线发现新的字段缺口时，必须在通讯中明确要求主线补入口径或计划。

## 6. P11.12.5 新增后端补充缺口

| 编号 | 事项                          | 前端用途                        | 需要后端或会议确认                                                                                                                                                           |
| ---- | ----------------------------- | ------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| G10  | 市场列表首屏加载真实口径      | `home/home` 市场列表首次进入    | 待定：普通藏品和盲盒首屏到底返回全量、当前页还是推荐窗口；请求字段是否固定为 `page/list_rows`；响应是否稳定包含 `total/data/list/hasMore`；空列表、失败和降级响应样例。      |
| G11  | 市场列表刷新/筛选重载真实口径 | 下拉刷新、系列筛选、类型切换    | 待定：刷新、系列筛选、类型切换或排序变化时是否重置分页；后端是否按同一过滤条件返回；请求快照、空结果、旧请求失效和返回顺序如何保证。                                         |
| G12  | 市场列表触底分页加载真实口径  | footer 加载更多和懒加载视觉状态 | 待定：普通藏品和盲盒是否都稳定支持触底分页；下一页字段、末页判断、无新增结果、`total` 与实际返回数量关系；旧前端普通藏品未证明真实分页追加，不能直接推导为当前后端分页能力。 |
| G13  | 市场列表排序真实口径          | 最新、价格、流通量等市场排序    | 待定：后端是否支持排序参数、默认排序、价格空值、上架时间空值、流通量字段、同价同时间稳定排序和分页一致性；未确认前不得宣称具备后端统一排序或全市场跨页全局排序。             |
