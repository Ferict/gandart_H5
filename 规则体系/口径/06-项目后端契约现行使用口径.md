## 3.5 当前不算后端已支持的内容

以下内容不得从本口径推导成“后端已支持当前 UI”：

1. 当前 UI 没有任何位置需要展示“成交量”；即使后端或旧前端存在交易、成交、流通字段，也不自动进入当前展示需求。
2. 链上溯源字段不算已签收：`owner`、`contract`、`chain`、`tokenStandard`、`provenance`。
3. 多维属性字段不算已签收：`traits`。
4. 只出现在 `unresolved.md`、`contracts/inferred/*` 或缺真实响应样例的字段，不算已支持。
5. 市场列表首屏加载、刷新/筛选重载、触底分页加载和排序的真实后端口径尚未签收；旧前端证据只能说明部分页面行为，不能直接推导成当前后端已经支持全量返回、统一分页或跨页全局排序。

4. 判断真实 API 是否已落地时，必须查真实 API 接口落地链，不查本口径文件。

## 4. 未确认项的明显标识规则

当前仓内只要引用到以下来源，就必须明显标注“待确认”：

1. `api-inventory.md` 中 `exported-unused` 的条目
2. `unresolved.md`
3. `contracts/inferred/*`
4. 缺真实样例、缺稳定消费证据、缺后端签收的字段

允许的明显标识写法固定为：

1. `待确认`
2. `未签收`
3. `仅推断`
4. `仅候选，不可直接接 UI`

禁止写法：

1. “后端已有”
2. “默认可用”
3. “按推断直接接”
