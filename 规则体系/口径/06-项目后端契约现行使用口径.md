## 3. 当前执行边界

当前仓的执行边界固定为：

1. `P11.12` 负责把现有 UI 适配到这套项目级后端契约。
2. Gemini UI 二线程必须读取这套契约，但不负责改造正式契约层。
3. 当前前端已落地内容域实现真值仍是 `src/contracts/content-api.contract.ts`。
4. 在 `P11.12` 完成前，项目级后端契约与当前前端实现契约允许并存，但必须由 adapter / runtime / service 吸收差异。

## 4. 未确认项的明显标识规则

当前仓内只要引用到以下来源，就必须明显标注“待确认”：

1. `api-inventory.md` 中 `exported-unused` 的条目
2. `unresolved.md`
3. `contracts/inferred/*`
4. 缺真实样例、缺稳定消费证据、缺后端签收的字段

允许的明显标识写法固定为：

1. `待确认`
2. `未签收`
3. `仅推断`
4. `仅候选，不可直接接 UI`

禁止写法：

1. “后端已有”
2. “默认可用”
3. “按推断直接接”

## 5. 当前二线执行口径

Gemini UI 二线程当前必须：

1. 把本目录和 `规则体系/后端契约/README.md` 作为后端契约入口。
2. 只消费 confirmed 项。
3. 对未确认项只做缺口说明，不在模板和样式层硬接。
4. 一旦字段不够，停在 `adapter / runtime / service` 边界，交给 `P11.12`。

## 6. 当前禁止事项

1. 不得再把 `01-正式契约/` 当作现行入口。
2. 不得把未确认项直接写入正式 view-model。
3. 不得绕过本口径和 `规则体系/后端契约/README.md` 自行拼装“新版后端结构”。
