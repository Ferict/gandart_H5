## 3. 当前有效口径

### 3.1 页面级热区例外清单

以下页面级例外当前仍有效，后续如继续存在，统一以本节为准：

1. `src/pages/home/components/HomeRailHomePanel.vue`
   - 市场标签横滑轨道的横向手势接管
   - 市场排序浮层的 `dismiss layer`
2. `src/pages/home/components/HomeRailActivityPanel.vue`
   - 公告标签横滑轨道的横向手势接管
3. `src/pages/home/components/HomeRailProfilePanel.vue`
   - 一级分类横滑轨道的横向手势接管
   - 二级分类横滑轨道的横向手势接管
4. `src/pages/home/components/HomeActivityDateFilterSheet.vue`
   - 日期筛选面板 `343px` 最大宽度
   - 七列等分日历网格公式

### 3.2 例外处理规则

1. 例外区仍必须保证真实热区满足“最短边 >= 44px”。
2. 横滑轨道只允许在确认横向意图后局部接管，不能再用无条件 `touchstart/touchmove stop` 抢主滚动。
3. 浮层 `dismiss layer` 只负责点外关闭，不得扩展成新的视觉热区语义。
4. 结构值、品牌装饰、小字号、搜索壳与输入框不属于热区例外；能修就直接修，不登记在本清单。

### 3.3 当前可用窗口宽度执行基线

1. 当前页面实现不再按 `H5 / Android / iOS` 分端建立适配责任矩阵，而是以运行时可用窗口宽度作为页面可用性基线。
2. 当前适配主线固定为“可用窗口宽度 + 轨道模式”，不按设备名硬分端。
3. 可用窗口宽度基线必须保证内容可读、关键操作可达、主滚动稳定、横滑与局部浮层不破坏主链交互。
4. 宽度分档与轨道模式必须通过结构和布局解决，不得靠页面私有补丁、空白堆叠、绝对定位硬凑。
5. 同一页面在不同可用宽度下必须保持语义结构稳定，再调整内容密度、装饰强度、局部排列或导航呈现方式。
6. 尺寸 token、视觉变量和 `px` 落地值都只是实现工具，不得反过来覆盖“可用窗口宽度 + 结构轨道”的主线。
7. 当前首页壳层只承认 `single-page` 与 `single-page-with-nav` 两类现行轨道；不恢复旧的普通宽屏多页并行、`double-page`、`triple-page` 或 `triple-page-with-stage` 方案。
8. `safe-area`、视口高度、键盘弹起、滚动容器、热区、小字号与视觉降级仍按 [08-交互与适配专题细则.md](/H:/工作/天异/uniapp+vue新架构/规则体系/08-交互与适配专题细则.md) 执行，但它们是窗口可用性和结构稳定性要求，不再作为独立“多端适配项目”推进。
9. 平台差异只作为环境补偿或降级处理，不新增按平台拆分的页面设计源、问题矩阵或专项看板。
10. 旧的“问题矩阵”和“视觉专项议题清单”只保留为历史输入和历史追溯材料，不再承担现行入口职责。
11. 新的热区例外或 UI 例外，如仍属于“当前执行口径”，必须写入本文；如已上升为长期稳定边界，则应写回规则正文。

## 4. 当前明确不扩展范围

1. 当前不新增新的旧式“专项档案专题口径文”。
2. 当前不把 UI 视觉优化议题清单继续作为执行看板。
3. 当前不把多端适配问题矩阵继续作为现行决策表。
4. 当前不按 `H5 / Android / iOS` 分端建立新的兼容验证矩阵或实修主线。
5. 当前不以设备名、尺寸 token 名称或旧宽屏多页方案作为页面适配决策源。
6. 当前不把已进入 `垃圾桶/` 的旧规则路径恢复成现行回链；旧口径只能经裁决后抽取并写入本文或规则正文。

## 5. 何时必须同步更新本文

命中以下任一情况时，必须同步更新本文：

1. 新增或撤销页面级热区例外。
2. `HomeRailHomePanel`、`HomeRailActivityPanel`、`HomeRailProfilePanel` 或 `HomeActivityDateFilterSheet` 的例外处理方式发生变化。
3. 可用窗口宽度现行执行基线发生调整，但尚未上升为长期稳定规则。
