# 08-P11.12 前端改造后口径

> 文档类型：现行口径
> 状态：active
> 更新时间：2026-04-25
> 说明：本文件是当前项目的“前端改造后口径”唯一入口，只回答 P11.12 改造后当前前端 UI 和 view-model 应该消费什么。它不是规则正文，不判断后端是否已经支持，也不承载真实 API path、method、DTO 或联调说明。

## 1. 职责边界

本文件负责定义前端目标态：

1. 当前页面最终消费的 view-model 语义。
2. adapter / runtime / service 应该给 UI 提供什么稳定字段。
3. 哪些字段不得进入当前 UI 稳定依赖。

本文件不做：

1. 不宣称后端已经支持。
2. 不列完整后端接口清单。
3. 不承载真实 API 接口；真实 API 仍按 [05-接口与联调规则.md](/H:/工作/天异/uniapp+vue新架构/规则体系/05-接口与联调规则.md) 落地。
4. 不替代 [06-项目后端契约现行使用口径.md](/H:/工作/天异/uniapp+vue新架构/规则体系/口径/06-项目后端契约现行使用口径.md)。
5. 不替代 [09-P11.12需后端补充口径.md](/H:/工作/天异/uniapp+vue新架构/规则体系/口径/09-P11.12需后端补充口径.md)。

## 2. 总原则

1. 当前 UI 不改视觉。
2. 页面模板不直接消费后端 DTO。
3. 后端差异由 `implementation -> adapter -> runtime/service -> view-model` 吸收。
4. UI 只消费稳定 view-model。
5. 当前没有任何页面需要展示“成交量”。

## 3. 当前页面字段目标

| 页面或区域                      | 前端改造后稳定语义              | 明确不使用                                                           |
| ------------------------------- | ------------------------------- | -------------------------------------------------------------------- |
| `home/home` 市场藏品卡左下角    | 总量/限量                       | 成交量、交易量、流通量、全平台持有人数                               |
| `home/profile` 个人藏品卡左下角 | 单个用户的持有量                | 市场总量、发行总量、流通量、全平台持有人数                           |
| `profile-asset-detail` 详情卡   | 持有、入库时间、版本、总量/限量 | 成交量、交易量、流通量                                               |
| 价格前缀                        | `￥`                            | `元` 作为价格前缀                                                    |
| 链上溯源                        | 当前不进入稳定依赖              | 未确认的 `owner`、`contract`、`chain`、`tokenStandard`、`provenance` |
| 多维属性                        | 当前不进入稳定依赖              | 未确认的 `traits`                                                    |

## 4. 字段解释

1. “总量/限量”表示该藏品发行或限定总量。
2. “总量”在当前 UI 里等同“限量”，不得解释成成交量。
3. “单个用户的持有量”只表示当前用户持有该藏品的数量。
4. “入库时间”表示该用户或该资产进入当前资产列表/库存链路的时间，具体来源待后端确认。
5. “版本”表示藏品详情展示需要的版本文本或版本枚举，具体来源待后端确认。

## 5. Adapter 输出要求

adapter 后续必须把后端字段归一成前端稳定语义：

1. 市场卡输出“总量/限量”语义，不输出成交量语义。
2. 个人藏品卡输出“当前用户持有量”语义，不复用市场统计字段。
3. 详情页输出“持有、入库时间、版本、总量/限量”语义。
4. 交易量、成交量、流通量、全平台持有人数只能保留在 backend normalized model 或待定字段中，不能默认进入 UI view-model。
5. 链上溯源和多维属性如需临时展示，只能标为 UI preset 或 fallback，不能写成后端稳定字段。

## 6. 二线使用规则

Gemini UI 二线程如需要字段，只能按本文件描述用途：

1. 需要市场卡左下角时，写“总量/限量”。
2. 需要个人藏品卡左下角时，写“用户持有量”。
3. 需要详情卡时，写“持有、入库时间、版本、总量/限量”。
4. 需要链上溯源或多维属性时，写入待确认缺口，不得自行接字段。

## 7. 市场加载与排序待定边界

当前前端改造后只承诺 UI 消费稳定列表 view-model 和稳定列表状态，不承诺后端真实加载或排序能力已经确定。

待后端确认前，以下内容均不得写成稳定 UI 依赖或后端已支持事实：

1. 市场列表首屏加载到底是全量返回、当前页返回还是推荐窗口返回。
2. 下拉刷新、系列筛选、类型切换或排序变化时，后端是否重置分页并按同一口径返回。
3. 触底分页加载是否对普通藏品和盲盒都稳定支持，以及末页、无新增和总数语义。
4. 最新、价格、流通量或其他排序是否由后端统一排序、前端仅对已加载集合排序，还是需要新的后端参数。

前端可以在 adapter / runtime / service 内做临时降级和 fallback，但页面模板不得直接消费后端分页协议、排序参数或旧前端 wrapper 形态。
