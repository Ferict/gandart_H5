# P11.12 后端接口接入分级清单

> 文档类型：后端 handoff 维护文档  
> 状态：active  
> 更新时间：2026-04-25  
> 说明：本文件给后端、前端主线和二线共同查看，用于说明当前页面接入新版后端契约时，哪些接口可以直接接、哪些建议后续优化、哪些必须后端补充、哪些需要新增能力。它不是正式 API 契约正文，真实 API 仍以 `规则体系/后端契约/` 和 `src/contracts/content-api.contract.ts` 为准。

## 1. 维护规则

本文件只维护四类：

1. `原接口接入`
   - 后端已有 confirmed wrapper，前端可以通过 `backend-http -> adapter -> content service/runtime -> view-model` 接入。
2. `建议改造`
   - 当前可以接，但字段结构、分页、状态或响应样例不够长期稳定，建议后端后续优化。
3. `需改造`
   - 当前契约证据不足，或字段语义与前端目标冲突；未确认前不得进入稳定 view-model。
4. `新增`
   - 当前没有足够的 wrapper 或能力，需要后端新增接口、字段、响应样例或业务口径。

本文件不做：

1. 不替代 `规则体系/后端契约/README.md`。
2. 不替代 `src/contracts/content-api.contract.ts`。
3. 不让页面直接消费后端 wrapper。
4. 不把待确认字段写成后端已支持。

## 3. 原接口接入

| 前端用途      | 后端 wrapper                 | 证据状态                       | 接入方式                            | 注意                                                                 |
| ------------- | ---------------------------- | ------------------------------ | ----------------------------------- | -------------------------------------------------------------------- |
| 首页 banner   | `banner.getBannerList`       | confirmed                      | 接入 `home` scene adapter           | 响应 item 结构需 adapter 兜底                                        |
| 首发/日历藏品 | `collection.getCalenderList` | confirmed，字段 Level B        | 接入 `home` scene adapter           | 可用 `id/batch_id/goods_type/listimg/name/publisher/total_num/price` |
| 普通市场列表  | `market.getMarketList`       | confirmed，字段 Level B        | 接入 `market_item` list adapter     | UI 只展示总量/限量，不展示成交量                                     |
| 盲盒市场列表  | `market.getMarketBoxList`    | confirmed，字段 Level B        | 与普通市场统一 adapter              | 兼容 `data[]` 包装                                                   |
| 公告列表      | `notice.getNoticeList`       | confirmed，响应 `data` Level B | 接入 `notice` list adapter          | item 字段未完全展开，需样例增强                                      |
| 用户基础信息  | `user.getUserInfo`           | confirmed                      | 接入 `profile` scene adapter        | 证据稳定到 `id/mobile`，不承诺资产统计                               |
| 邀请配置候选  | `common.getConfig`           | confirmed，`body.key` Level A  | 接入 profile adapter                | key 和返回结构必须白名单化                                           |
| 抽签详情基础  | `draw.getDraw`               | confirmed，`body.id` Level A   | 接入 `priority-draw` detail runtime | 稳定字段只有部分时间、剩余量、说明                                   |
| 抽签动作      | `draw.draw`                  | confirmed                      | 接入抽签动作 runtime                | 返回 primitive string，结果结构不稳定                                |
| 抽签记录      | `draw.drawRecord`            | confirmed                      | 接入结果/记录 runtime               | 数组 item 结构需样例                                                 |
| 抽签日志      | `draw.drawLog`               | confirmed                      | 接入记录 runtime                    | 数组 item 结构需样例                                                 |

## 4. 建议改造

| 前端用途 | 当前可接方式                                       | 建议后端优化                                        | 原因                                                 |
| -------- | -------------------------------------------------- | --------------------------------------------------- | ---------------------------------------------------- |
| 市场分页 | `market.getMarketList` / `market.getMarketBoxList` | 明确 `page/list_rows/per_page/total/data/list` 语义 | 前端当前统一是 `page + pageSize + total + items`     |
| 市场价格 | 市场 wrapper 价格字段                              | 明确价格单位是分、元、字符串还是整数                | 前端价格符号固定 `￥`，数值格式不能猜                |
| 公告列表 | `notice.getNoticeList`                             | 给出公告 item 样例、分类字段、未读字段              | 当前只稳定到 `data`                                  |
| 活动入口 | `news.getActivityList`                             | 补活动 item 字段样例                                | 当前稳定证据偏弱，只能辅助活动入口                   |
| 抽签页面 | `draw.*` 多 wrapper 组合                           | 拆清列表、详情、报名/抽签、结果、记录               | 当前二线页面需要的是完整活动页模型，不是单个详情请求 |
| 配置读取 | `common.getConfig`                                 | 明确邀请相关 key 与返回结构                         | 防止配置被任意字符串调用                             |

## 6. 新增

| 新增项                       | 原因                                                                        | 建议                                                |
| ---------------------------- | --------------------------------------------------------------------------- | --------------------------------------------------- |
| `backend-http` provider      | 当前只有 `mock` 和 `/api/content/*` 的 `http`，没有旧 wrapper 接驳 provider | 前端新增 implementation，不要求后端新增聚合接口     |
| 抽签活动列表能力             | `priority-draw` 需要列表页，`draw.getDraw(id)` 只适合详情                   | 后端提供活动列表，或确认可由现有活动列表 + 详情组合 |
| 抽签资格只读能力             | 页面需要展示“是否符合资格”和原因文案                                        | 若不新增接口，则后端确认可复用哪个 wrapper 和响应   |
| 抽签结果结构                 | 结果弹层需要中签/未中/异常、奖品组合、行动文案                              | 后端补正式响应样例                                  |
| 资产总额字段或算法           | 个人中心要展示资产总额                                                      | 后端提供字段，或确认前端按地板价/持有量派生         |
| service reminder action 归属 | 当前 `service-reminder-consume` 只有 content action                         | 后端确认旧 wrapper 归属或本期继续保留当前 action    |

启动方式：

1. `VITE_CONTENT_PROVIDER=backend-http`
2. `VITE_CONTENT_BACKEND_API_BASE_URL=https://后端域名`
3. 若没有配置 `VITE_CONTENT_BACKEND_API_BASE_URL`，会回退读取 `VITE_CONTENT_API_BASE_URL`。

未接入项必须继续显式返回未支持 envelope，不能把 mock 或空数据伪装为后端成功。

## 8. 加载与分页

加载本身不新增后端接口。

后端只需保证列表接口可被 adapter 归一为：

1. `page`
2. `pageSize`
3. `total`
4. `items`

成功但无数据时必须能归一为：

1. `items: []`
2. `total: 0`

前端继续负责：

1. loading 动画
2. 自动重试
3. 无进展重试
4. 请求超时
5. 旧请求失效
6. 单飞和分页触发冷却

## 9. 维护要求

每次发生以下变化，都必须更新本文件：

1. 后端确认新字段或新样例。
2. 前端 adapter 完成一个页面域接入。
3. 某个待确认项被用户批复为本期不做。
4. 二线页面新增字段缺口。
5. `backend-http` provider 接入范围扩大。
