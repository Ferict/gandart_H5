# 源码交付白名单

> 文档类型：交付边界说明
> 状态：active
> 更新时间：2026-04-20
> 说明：本文件约束可运行完整 CI 的 GitHub 联调源码仓允许包含哪些文件和目录，以及哪些内容必须排除。

## 1. 文件定位

本文件用于约束“GitHub 联调源码仓”的交付边界。

GitHub 联调源码仓当前定位是“可运行完整 CI 的源码仓”：

1. 可运行的前端源码仓。
2. 可供后端查看真实调用链的联调仓。
3. 带完整质量规则、文档检查入口和 CI 配置的工程仓。

它不是：

1. 只放 API 文档的 handoff 包。
2. 只保留最小源码的裁剪包。
3. 垃圾桶、备份和本地素材仓。

如果后续需要“只给后端看的最小 handoff 包”，应另建最小移交清单；不能用本文件裁剪后仍要求当前 `quality-gate` 原样通过。

## 2. 交付原则

1. 交付仓以“前端完整源码 + 必要工程配置 + 后端联调说明”为主。
2. 交付仓必须保留 `src/`、`tests/`、`scripts/`、`.github/` 和根工程配置。
3. 当前唯一正式 API 真值源 [src/contracts/content-api.contract.ts](./src/contracts/content-api.contract.ts) 必须随源码交付。
4. `backend-handoff/` 和 `references/` 是后端联调阅读入口和参考索引，必须随源码交付。
5. 当前 CI 和本地质量门禁直接依赖的质量资产、规则入口和 active 留档必须随源码交付。
6. 历史材料、垃圾桶内容、构建产物、本地依赖和本地工具产物不得随源码交付。

## 4. 允许交付的目录

以下目录允许进入 GitHub 联调源码仓：

```txt
.github/**
规则体系/**
backend-handoff/**
references/**
scripts/**
promptfoo/**
semgrep/**
vale-styles/**
.continue/**
src/**
tests/**
留档/README.md
留档/审查记录/*.md
留档/计划记录/*.md
留档/交付记录/*.md
留档/即时思路记录/*.md
提交清单/README.md
垃圾桶/README.md
```

目录说明：

1. `src/**`：当前前端正式源码。
2. `tests/**`：单元测试与 E2E 测试。
3. `scripts/**`：工程脚本、联调验证脚本和质量脚本。
4. `.github/**`：GitHub Actions 和仓库协作说明。
5. `规则体系/**`：当前文档 lint 和仓内规则阅读入口依赖的规则正文。
6. `backend-handoff/**`：后端 handoff、联调环境和验证说明。
7. `references/**`：最小参考索引，不保存接口代码快照副本。
8. `promptfoo/**`、`semgrep/**`、`vale-styles/**`、`.continue/**`：当前 CI 和本地质量门禁依赖的规则、评测和检查资产。
9. active `留档/**`：当前 `lint:docs` 与 `lint:format` 覆盖的正式计划、审查、交付和即时思路记录。
10. `提交清单/README.md`、`垃圾桶/README.md`：当前文档检查需要的目录边界说明；只交 README，不交目录内部历史或垃圾内容。

## 5. 明确不交付内容

以下目录或文件不得进入 GitHub 联调源码仓。

说明：第 4 节已经明确允许的 `提交清单/README.md` 和 `垃圾桶/README.md` 是例外，只保留目录边界说明，不保留目录内部历史或垃圾内容。

```txt
留档/历史记录/**
审核与修改计划/**
对话台账/**
提交清单/**
项目索引/**
项目台账.md
垃圾桶/**
备份/**
dist/**
node_modules/**
.codex-tmp/**
.tools/**
test-results/**
playwright-report/**
coverage/**
公开图库候选/**
新藏品/**
.tmp-l4-devserver.log
```

排除原因：

1. `留档/历史记录/**`、旧台账、旧审核计划、旧索引和备份目录都不是当前 CI 必需输入。
2. `垃圾桶/**` 内部内容不是现行入口；只允许保留 `垃圾桶/README.md` 说明隔离边界。
3. `提交清单/**` 内部内容不是当前交付输入；只允许保留 `提交清单/README.md` 说明边界。
4. `新藏品/**` 和候选图库属于本地素材源，不是运行时正式资源目录。
5. `dist/**`、`node_modules/**`、`test-results/**`、`playwright-report/**`、`coverage/**` 是构建、依赖或测试产物。
6. `.tools/**` 是本地下载的辅助工具产物，不是源码仓必需输入。

## 6. 打包执行规则

1. 打包时以白名单为准，不做黑名单反推。
2. 如需新增交付文件或目录，必须先更新本白名单。
3. GitHub `main` 若承担后端联调源码仓角色，必须按本白名单校验。
4. 不得靠口头说明替代交付清单。
5. 不得把旧 API 正文、旧接口契约目录或历史模板恢复成现行交付入口。

## 7. 最小验收

交付前至少确认：

1. `src/` 完整，且包含 [src/contracts/content-api.contract.ts](./src/contracts/content-api.contract.ts)。
2. `tests/`、`scripts/`、`.github/` 和根工程配置完整。
3. `AGENTS.md`、`规则体系/`、active `留档/`、`提交清单/README.md`、`垃圾桶/README.md` 已带上，保证文档门禁可运行。
4. `backend-handoff/` 与 `references/README.md` 已带上。
5. `promptfoo/`、`semgrep/`、`vale-styles/`、`.continue/` 已带上，保证质量门禁可运行。
6. `留档/历史记录/`、`垃圾桶/` 内部旧内容、旧台账、旧审核计划、旧索引、备份目录未带上。
7. `node_modules/`、`dist/`、测试产物、覆盖率产物和 `.tools/` 未带上。
8. `新藏品/`、候选图库和本地原图素材未带上。
