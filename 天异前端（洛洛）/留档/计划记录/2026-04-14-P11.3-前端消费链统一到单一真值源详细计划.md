# 1. 目标

`P11.3` 只做一件事：

把 `src/` 内的正式内容域 API 消费链统一到 [content-api.contract.ts](/H:/工作/天异/uniapp+vue新架构/src/contracts/content-api.contract.ts) 这一份真值源。

# 2. 目标状态

`P11.3` 完成后，前端侧必须满足：

1. `content.http.ts` 不再硬编码 5 个正式 path。
2. `content.port.ts` 与 `content.service.ts` 只承接 contract，不再补第二套接口定义语义。
3. 任何新增正式接口路径、方法、request/response 变更，都必须先改 contract 文件，前端代码才能跟着变。
4. `src/` 内不存在“代码看 contract，路径看 adapter，字段归属看另一份文档”的并行理解方式。
