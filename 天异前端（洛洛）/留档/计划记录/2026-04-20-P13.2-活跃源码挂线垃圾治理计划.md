## 1. 目标

1. 修正“挂到就算有用”的错误审查口径。
2. 把已挂线但无真实页面、无真实业务、无用户确认目标的代码列为治理候选。
3. 先拆线、再验证、再按计划转移到 `垃圾桶/`，不直接删除。
4. 区分“垃圾候选”和“需要重构但仍支撑当前主流程”的代码。

第一批候选：

1. `src/pages/settings/`
2. `src/pages/service-entry/`
3. `src/pages/action-entry/`
4. `src/pages/content-resource/`
5. `src/pages/profile-address/`
6. `src/pages/profile-assets/`
7. `src/pages/notice-detail/`
8. `src/pages/updating/`
9. `src/services/page-entry/`

11. `src/adapters/pageContent.mock.adapter.ts`
12. `src/models/route-query/` 中只服务上述占位入口的 query 类型和 schema

不纳入第一批：

1. `src/pages/home/`
2. `src/pages/profile-asset-detail/`
3. `src/contracts/content-api.contract.ts`
4. `src/services/content/`
5. `src/ports/content.port.ts`
6. `src/implementations/content.*`
7. `src/mocks/content-db/`

说明：

1. 首页内部过度拆分属于后续简化重构，不是本批垃圾桶转移对象。
2. 内容域 mock/http 链是当前未接真实后端时的运行基础，不按垃圾处理。

## 3. 已确认方向

1. 不保留独立 `settings`、`service-entry`、`action-entry`、`content-resource`、`profile-address`、`profile-assets`、`notice-detail` 页面壳。
2. 未完成功能入口统一进入 `updating`。
3. `updating` 保留为统一建设中承载页，通过 query 文案适配不同入口。
4. notice placeholder 内容链退出当前路径；正式内容域公告接口链保留。
5. 退场源码转入用户确认的 `垃圾桶/P13.2-挂线占位入口/` 本地冻结区。

### 4.1 第一批：拆导航线

1. 修改 `homeRailNavigation.service.ts`，不再把不保留的入口构造成独立页面 URL。
2. 对仍需要提示未开放的点击行为，统一返回 `updating` 或在当前页面内处理。
3. 不再生成独立占位页 URL。

1. 新增共享建设中组件 `ConstructionPlaceholder.vue`。
2. 将 `pages/updating/index.vue` 改成路由适配层，不再直接承载建设中 UI 细节。
3. 该先导子批未拆其他页面入口，后续已由本次独立执行计划继续完成拆线和转移。

执行结果：

1. 已完成；旧导航函数名保留，返回目标统一改为 `updating`。
2. `updating` 文案解析已收口到页面附近。

检查：

1. `npm run type-check`
2. `npm run test:unit`

### 4.2 第二批：拆页面注册

### 4.3 第三批：转移候选文件

目标：

1. 将已拆线、已不注册、已无引用的页面目录转移到 `垃圾桶/`。

3. 如 `垃圾桶/` 当前结构无法承接，停止并请用户确认落点。

硬约束：

1. 不执行物理删除。

检查：

1. `npm run type-check`
2. `npm run test:unit`
3. `npm run knip`

2. 转移前已校验源路径与目标路径均在当前仓和指定垃圾桶范围内。

### 4.4 第四批：精简 query 与测试

目标：

1. 删除已无入口使用的 route query 类型和 schema 分支。

2. 共享 `route-query` 旁路线已退出当前路径。

## 5. 停止条件

出现任一情况必须停止：

1. 用户未确认某个页面入口是否保留。
2. 需要新建 `垃圾桶/` 长期分类目录。
3. 拆线会影响首页主流程、藏品详情页或正式内容接口链。
4. `type-check`、`test:unit`、`knip` 或构建失败且无法在本批范围内定位。
5. 发现某个候选入口其实承载了真实用户要求或真实业务目标。

## 6. 验收标准

1. 所有已处理对象都先拆线、再转移，没有直接删除。
2. `pages.json` 不再注册用户确认不保留的占位页面。
3. `homeRailNavigation.service.ts` 不再生成人工保活的垃圾候选 URL。
4. `page-entry` 和 `pageContent` 旁路线只保留真实需要部分，或整体转入 `垃圾桶/`。
5. 必要检查通过，并形成交付记录和 Git 提交。
