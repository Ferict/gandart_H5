## 1. 目标

1. 把当前仓全部现行 UI 对象列全。
2. 给每个对象固定 `unitId`、层级、优先级、敏感性和依赖面。
3. 为 `P11.10.5` 的多子代理 ownership 划分提供唯一真值。

## 2. 固定方法

1. 只把 `.vue` 视为 UI 治理对象。
2. `runtime / composables / helpers / services` 不单列为 UI unit，只挂到所属对象。
3. 当前仓所有未进显式对象清单的 `.vue` 都视为治理缺口，不允许口头补。

## 3. 产物

1. 全量治理对象清单
2. 高优先级对象清单
3. 敏感对象清单
4. 依赖面挂接规则

## 4. 下游使用方式

1. `P11.10.3` 按本清单抽取消费契约。
2. `P11.10.4` 按本清单建立壳层映射。
3. `P11.10.5` 按本清单划分 worker ownership。

## 5. 验收标准

1. 当前仓无游离 `.vue`。
2. 所有正式页面与共享组件都有 `unitId`。
3. 已能支持多子代理并行 ownership 划分。
