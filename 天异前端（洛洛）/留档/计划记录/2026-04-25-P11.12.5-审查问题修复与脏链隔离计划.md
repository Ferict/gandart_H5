# P11.12.5 审查问题修复与脏链隔离计划

## 目标

修复 `backend-http` 真实接驳风险，同时把二线 UI 脏链和未跟踪来源包隔离清楚。原则固定：不改视觉、不改正式 `content-api.contract.ts` 形态、不提交二线 UI、不删除任何文件。

## 执行范围

1. 修复 `home/activity/profile` 场景聚合业务错误处理。
2. 修复空首发伪造 `0` 价格业务块的问题。
3. 在 `backend-http` adapter 内对市场当前页做 `listedAt / priceInCent` 稳定排序，不向旧后端发送未确认排序参数。
4. 严格校验 `VITE_CONTENT_PROVIDER`，未知 provider 不允许静默回落 mock。
5. 修复既有审查记录格式门禁问题。
6. 在需后端补充口径中记录“市场跨页全局排序参数缺口”。
7. 提交隔离：不纳入 `SecondaryPageTopbar.vue`、`HomeRailTopbar.vue`、`src/pages/updating/**`、`来源源码/`。

## 停止条件

1. 需要修改正式 `src/contracts/content-api.contract.ts`。
2. 需要改视觉、布局、topbar 外观或二线 UI 产物。
3. 需要删除、清理、物理移除或 `git rm` 文件。
4. 需要真实后端账号、base URL 或样本数据才能继续。
5. 提交边界无法只包含本批文件。

1. `git diff --check`

## 完成标准

1. 新增/更新单测覆盖场景子接口业务失败、空首发、未知 provider、市场当前页排序。
2. 质量门禁恢复通过，或把非本批阻塞明确记录。
3. 本批交付记录写明实际修改、验证结果、提交索引和仍隔离的脏链。
