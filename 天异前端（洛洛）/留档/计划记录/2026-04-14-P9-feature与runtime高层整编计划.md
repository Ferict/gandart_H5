# 2026-04-14 P9 feature 与 runtime 高层整编计划

> 文档类型：计划记录
> 状态：active
> 执行状态：已完成
> 更新时间：2026-04-14

## 1. 目标

`P9` 承接 `P7/P8` 之后仍然保留的高层结构问题，目标固定为：

1. 把当前“目录可解释、物理落点基本正确”的状态，推进到“高层 feature/runtime 关系也可解释”。
2. 为后续更大层级的结构调整提供蓝图和批次顺序，而不是继续零散补丁。
3. 在不引入新的 `src/` 顶层目录前提下，先澄清 `pages / services / models / 接入链` 的长期边界。

## 2. 固定边界

本批只允许：

1. 做高层结构审查和蓝图定稿。
2. 在现有锚点下做少量低风险归位。
3. 回写目录说明、留档和计划索引。

本批明确不做：

1. 直接新建 `src/features/` 或其他新的 `src/` 顶层目录。
2. 发起大范围跨锚点物理迁移。
3. 改动后端接口契约、provider、缓存协议或刷新协议。

## 3. 默认决策

以下默认决策在 `P9` 内生效：

1. `pages/*` 继续作为页面壳与 page-local runtime 的首要锚点。
2. `services/*` 继续只承载真正跨页面、跨领域的运行时服务，不回流页面私有装配。
3. `contracts / ports / implementations / adapters` 继续视为内容接入链，本批只做语义澄清，不做拆并。
4. 如后续需要引入新的长期目录类型或新的 `src/` 顶层组织，先形成明确蓝图，再单独请求许可。

## 4. 执行顺序

1. `P9.1` feature 与 runtime 高层结构基线审查
2. `P9.2` 高层蓝图与默认分层定稿
3. `P9.3` 低风险高层归位试点
4. `P9.4` 收口复核与下一阶段边界整理

## 5. 当前执行结论

1. `P9.1` 已完成：当前高层问题和继续治理的边界已经审清。
2. `P9.2` 已完成：高层蓝图、默认分层和后续试点边界已定稿。
3. `P9.3` 第一批已完成：`services/page-entry` 已定性为冻结过渡锚点，不做本批物理迁移。
4. `P9.3` 第二批已完成：`pages/home` 与 `services/home-rail` 的高层边界已经冻结，不做本批物理迁移。
5. `P9.3` 第三批已完成：`pages/profile-asset-detail` 与 `services/profile-asset-detail` 的高层边界已经冻结，不做本批物理迁移。
6. `P9.4` 已完成：主线收口复核已经确认当前批次可以闭环。
