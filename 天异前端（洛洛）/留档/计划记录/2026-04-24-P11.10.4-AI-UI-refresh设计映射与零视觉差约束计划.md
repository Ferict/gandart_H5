## 1. 目标

1. 固定视觉壳基线。
2. 固定 token 基线。
3. 固定零视觉差检查清单。
4. 为 `P11.10.5 ~ P11.10.7` 提供硬约束。

## 2. 固定做法

1. 不引入新的视觉 token 系统。
2. 不调整现有变量值。
3. 不调整任何 `<style>` 的可见输出。
4. 只做“登记现状、冻结现状、检查现状”。

## 3. 下游使用方式

1. `P11.10.5`
   - 结构收口时必须以本批检查清单为回归门槛。
2. `P11.10.6A / 6B`
   - 试跑时必须核对 shared/global 壳与详情页壳未发生可见差异。
3. `P11.10.7`
   - 全量收口时必须继续沿用同一份零视觉差清单。

## 4. 验收标准

1. 当前视觉壳基线已写清。
2. 当前 token 基线已写清。
3. 当前零视觉差检查清单已写清。
4. 后续代码批不再需要临时决定“哪些视觉不能动”。
