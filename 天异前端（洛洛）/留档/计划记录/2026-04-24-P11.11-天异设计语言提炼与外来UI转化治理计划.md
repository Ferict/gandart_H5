## 1. 目标

本计划只服务一个核心目标：建立天异自己的 UI 设计语言，并让后续所有外来 UI 都必须先翻译成这套语言，再进入前端实现。

本计划解决以下问题：

1. 当前新页面不能继续按外来 UI 或 Gemini 稿直接落地。
2. 后续 Figma、截图、React、Pencil、提示词等输入必须统一进入转化治理流程。
3. UI 转化必须保持与请求链、接口契约、provider、正式 DTO 解耦。
4. 后续 skill 只能从已经落盘的规则、口径和计划派生，不能成为第二套真值源。

## 2. 不做什么

本计划明确不做：

1. 不处理 `src/pages/updating/updatingRouteQuery.ts` 的 review finding。
2. 不把 Gemini 稿、Figma 稿、截图或 React 代码作为视觉真值源。
3. 不覆盖 `P11.10` 视觉冻结与零视觉差结构治理主线。
4. 不在本计划中改后端契约、provider、service 或正式接口链。
5. 不顺手改 `src/uni.scss`、`src/components/*`、`src/pages.json` 或 App 壳。
6. 不新建顶层 `docs/`、顶层 `.codex/` 或任何平行现行说明目录。
7. 不创建正式 skill。
8. 不绕过 [规则体系/后端契约/README.md](/H:/工作/天异/uniapp+vue新架构/规则体系/后端契约/README.md) 自行拼装“新版后端结构”。

## 3. 外部设计技能调研吸收

本计划吸收公开 GitHub 设计与 skill 方法中的可复用机制，但不照搬其目录结构或自由创作模式。

   - 吸收点：先定清晰审美方向，再写代码；避免通用 AI 感。

## 7. P11.11.2 天异设计语言矩阵

目标：形成可执行、可检查、可复用的天异 UI 语言。

矩阵按五类组织：

1. `components`
   - 页面骨架、导航栏、卡片、按钮、标签、热区、空态。
2. `variables`
   - 色彩、spacing、radius、shadow、safe area、宽度基准。
3. `styles`
   - 字体层级、字重、英文 micro label、图片裁切、状态表达。
4. `patterns`
   - 首屏视线顺序、主行动、次行动、列表、详情、活动、资产感表达。
5. `copy`
   - 中文主导、英文辅助、AI 味文案禁用、业务活动文案规则。

每个条目必须写清：

1. 推荐写法。
2. 禁止写法。
3. 现有样本。
4. 适用对象。
5. 验收方式。

初始裁决：

1. 推荐白色轻质 surface、浅灰页面背景、20/24 圆角、轻阴影、cyan 点状强调、真实内容优先、克制黑色 CTA。
2. 禁止大面积 AI 渐变海报、过多 900 字重、外来导航栏、外来 hover 风、无来源英文堆叠、装饰性图标污染信息层级。

验收标准：

1. 后续拿到任意 Gemini / Figma / 截图 / React 输入，都能按矩阵判断保留什么、丢弃什么、重建什么。
2. 矩阵不是审美描述，而是执行规则。

## 9. P11.11.4 外来 UI 转化 brief 与流程

目标：以后任何外来 UI 进入前端前，先填写转化 brief，不能直接改代码。

brief 必须包含：

1. 来源类型：Gemini / Figma / 截图 / React / Pencil / 文字描述。
2. 目标对象：`page-shell / panel / section / shared component / global component`。
3. 设计方向声明：克制、内容优先、资产感、活动感、工具感或其他已裁决方向。
4. 保留内容：信息架构、内容优先级、交互意图、状态需求。
5. 丢弃内容：外来字体、导航栏、渐变、阴影、卡片语法、AI 文案。
6. 天异翻译：对应设计语言矩阵的页面骨架、卡片、图片、CTA、热区。
7. 数据需求：当前 view model 已有字段、缺失字段、Level C 字段。
8. 写入集合：page-local 优先，shared/global 另立计划。
9. 视觉证据：改造前、改造后、与现有样本并排。

验收标准：

1. 没有 brief，不进入代码。
2. brief 没有完成“外来 UI -> 天异语言”翻译，不进入代码。
3. 出现新字段缺口，不在 UI 中硬造，停在 adapter / runtime / service 输入侧。

## 11. 三柱评审标准

每次视觉转化验收必须同时通过三柱评审：

1. `frictionless`
   - 用户第一眼看到的 3 个元素是否符合业务优先级。
   - 每个页面是否只有 1 个明确主行动。
   - 次行动是否没有抢主行动。
2. `quality craft`
   - 字体、颜色、圆角、阴影、间距是否可追溯到天异矩阵或现有样本。
   - 卡片、按钮、图片、标签是否属于同一设计语言。
   - 响应式宽度下是否仍然成立。
3. `trustworthy building`
   - 空态、错误态、长文案、图片失败、窄屏、安全区是否完整。
   - 接口缺口是否停在 view model / adapter / runtime 输入侧。
   - UI 是否继续与 transport-only 字段解耦。

## 12. 失败退出与回退

必须停止的情况：

1. 需要改 shared/global 才能继续。
2. 需要改 `src/uni.scss` 才能继续。
3. 需要改接口契约、provider、service、路由。
4. 外来 UI 要求改变核心交互语义。
5. 设计语言矩阵无法覆盖当前页面需求。
6. 截图验收显示新页面仍不像现有天异 UI。
7. Git 工作树中非本轮改动影响当前批次判断。

处置方式：

1. 页面问题回到 `P11.11.3` 修正。
2. 设计语言缺口回到 `P11.11.2` 补矩阵。
3. 流程缺口回到 `P11.11.4` 补 brief。
4. 共享或全局变更单独立计划，不在本批顺手做。
