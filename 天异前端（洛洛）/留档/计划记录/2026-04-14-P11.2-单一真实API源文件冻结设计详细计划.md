## 1. 目标

`P11.2` 只做一件事：

把 [src/contracts/content-api.contract.ts](/H:/工作/天异/uniapp+vue新架构/src/contracts/content-api.contract.ts) 冻结成**后端唯一正式阅读正文**，并为后续 `P11.3 ~ P11.6` 提供唯一上位真值。

## 2. 设计目标状态

`P11.2` 完成后，该文件必须满足：

1. 后端只看这一份文件，就能知道当前正式的 5 个接口是什么。
2. 后端只看这一份文件，就能知道每个接口的：
   - 方法
   - 路径
   - request DTO
   - response envelope + DTO
   - 当前正式范围
3. 后端只看这一份文件，就能知道：
   - 哪些字段是正式后端字段
   - 哪些仍是前端上下文字段
   - 哪些仍是 UI preset，不应要求后端实现
4. 该文件足以成为：
   - `content.port.ts`
   - `content.http.ts`
   - `content.service.ts`
     的唯一上位 contract 来源。

## 3. 固定结构

冻结后的 [src/contracts/content-api.contract.ts](/H:/工作/天异/uniapp+vue新架构/src/contracts/content-api.contract.ts) 结构固定为：

### 3.1 顶部总说明

必须明确：

1. 本文件是唯一正式 API 真值源。
2. 后端阅读顺序只需这一份文件。
3. 页面结构、刷新协议、缓存键、query snapshot 不属于本文件。

### 3.2 正式 endpoint 总表层

必须新增一个显式总表层，至少能表达：

1. `GET /api/content/scene`
2. `GET /api/content/resource`
3. `GET /api/content/list`
4. `POST /api/content/action/notice-read`
5. `POST /api/content/action/service-reminder-consume`

建议实现方式：

1. 以导出的 endpoint 常量对象或等价 TS 常量表达
2. 每项都带：
   - `method`
   - `path`
   - `request`
   - `response`
   - `summary`

### 3.3 正式 contract 基础层

保留并整理：

1. `ContentEnvelope<T>`
2. `ContentResourceType`
3. `ContentListResourceType`
4. 正式 `Scene / Resource / List / Action` request DTO
5. 正式 DTO 与 payload 结构

### 3.4 字段归属层

必须新增显式说明，写清：

1. 正式后端字段范围
2. 前端上下文范围
3. UI preset 范围
4. 当前明确不纳入 contract 的字段

至少必须直接点名：

1. `params.category / params.subCategory`
2. `route.query.*`
3. `query snapshot / cache key`
4. `owner / contract / chain / tokenStandard / traits / provenance`

### 3.5 当前正式边界层

必须写清：

1. 当前正式列表分页仍是 `page + pageSize`
2. `cursor` 仍不在正式范围
3. 当前正式 scene 只有：
   - `home`
   - `activity`
   - `profile`
   - `settings`
   - `service_hub`
4. 当前正式列表资源只有：
   - `market_item`
   - `notice`
   - `profile_asset`
