# 2026-04-13 G0.5 旧历史路径迁移顺序与保留边界

> 文档类型：执行记录  
> 状态：active  
> 更新时间：2026-04-13  
> 说明：本记录用于固定旧历史路径迁移到新历史记录体系时的顺序、目标落点和保留边界。

## 1. 盘点对象

当前纳入迁移规划的旧历史路径有：

1. `项目索引/`
2. `对话台账/`
3. `审核与修改计划/`
4. `项目台账.md`

当前盘点体量：

1. `审核与修改计划/`：`138` 个文件
2. `对话台账/`：`15` 个文件
3. `项目索引/`：`11` 个文件
4. `项目台账.md`：根文件 `1` 个

## 2. 迁移顺序

迁移顺序固定建议为：

1. `项目索引/`
2. `对话台账/`
3. `审核与修改计划/`
4. `项目台账.md`

## 3. 顺序理由

### 3.1 `项目索引/` 先迁

原因：

1. 体量小
2. 结构简单
3. 更容易先分出“仍有价值的历史索引”与“可直接废弃的旧索引”

### 3.2 `对话台账/` 第二迁

原因：

1. 目录结构简单
2. 基本按日期线性组织
3. 迁移时更适合先建立“对话历史”的新命名和回链规则

### 3.3 `审核与修改计划/` 第三迁

原因：

1. 体量最大
2. 子目录最多
3. 同时混有历史审查、历史计划、历史执行与专项档案
4. 需要最后再做细分，避免一开始就误搬

### 3.4 `项目台账.md` 最后迁

原因：

1. 它是旧体系里的全局总账
2. 迁移过早会丢失中间阶段的回链锚点
3. 应在其他历史路径都已有稳定新落点后，再重建成新的历史总账入口

## 4. 新落点规划

新历史记录体系的目标落点固定为：

1. `项目索引/` -> `留档/历史记录/索引与总账/`
2. `对话台账/` -> `留档/历史记录/对话/`
3. `审核与修改计划/` -> `留档/历史记录/批次档案/`
4. `项目台账.md` -> `留档/历史记录/索引与总账/`

## 5. 保留边界

在物理迁移前：

1. 这些旧路径继续只保留历史参考属性。
2. 新规则、新计划、新审查、新交付不得再写回这些旧路径。
3. 若必须引用旧历史内容，只能作为历史证据，不得当作现行规则真值源。

在物理迁移时：

1. 先做映射与验证
2. 再迁入新历史目录
3. 最后再决定旧容器是进入垃圾桶还是保留冻结快照

## 6. 当前不做的事

1. 不在本轮直接迁移旧历史文件
2. 不在本轮直接删除旧历史路径
3. 不在本轮直接把旧历史路径整体送入垃圾桶
