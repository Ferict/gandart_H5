## 3. 定稿结果

### 3.1 `src/` 长期逻辑蓝图

当前 `src/` 的长期逻辑蓝图固定为：

1. 页面入口与页面私有实现层
   - `pages/`
2. 共享 UI 与共享前端运行时基础层
   - `components / composables / models / stores / utils / static`
3. 领域共享运行时服务层
   - `services/*`
4. 内容接入链
   - `contracts / ports / implementations / adapters`
5. mock 与开发数据支撑层
   - `mocks`

### 3.2 顶层目录默认边界

#### 稳定锚点

1. `pages`
2. `services`
3. `models`
4. `mocks`
5. `static`

#### 稳定共享层

1. `components`
2. `composables`
3. `stores`
4. `utils`

#### 内容接入链

1. `contracts`
2. `ports`
3. `implementations`
4. `adapters`

约束：

1. 这四层当前保留。
2. 只允许继续承接内容接入或页面内容适配相关新增。
3. 不再把新的业务运行时、页面私有逻辑或试验件塞进这四个顶层目录。

#### 顶层新增冻结区

1. 本批之后不再新增新的 `src` 顶层技术目录。
2. 页面私有能力优先靠近 `pages/*`。
3. 跨页共享能力优先进入：
   - `services/*`

   - `components / composables / stores / utils`

#### 后续并轨候选

1. `contracts / ports / implementations / adapters`
   - 后续可作为单独高级治理主题做并轨收敛
2. `components / composables / stores / utils`
