## 2. 收口结果

1. `HomeRailActivityPanel` 不再直接持有下列高耦合装配链：
   - `useActivityScenePatchController(...)`
   - `useHomeRailActivityNoticeDataPipeline(...)`
   - `useHomeRailActivityViewState(...)`
   - `useActivityNoticeVisualReveal(...)`
   - `useActivityNoticeResultWindow(...)`
   - `useActivityPagePresentationRuntime(...)`
   - `useHomeRailActivityPresentationState(...)`
   - `useHomeRailActivityNoticeReload(...)`
   - `useHomeRailActivityContentLifecycle(...)`
   - `useHomeRailActivityEffects(...)`
2. 新 `useHomeRailActivityPanelRuntime` 统一承接：
   - notice scene patch
   - notice query 与 remote list pipeline
   - 结果区 view state / visual reveal / result window
   - search reveal / reload / navigation
   - page presentation runtime
   - refresh / activation / visible-update lifecycle
   - watcher/effects 装配
3. 父页继续保留：
   - `noticeResultsStageRef` 的 template-ref adapter
   - 模板结构与 `defineExpose`

## 3. 风险控制

1. 仍然只搬接线，不改活动 rail 既有运行时模块接口。
2. 模板中的 notice 类型文案和时间格式继续由 runtime 明确返回，不引入新的 shared 抽象。
3. `openDateFilter` 事件继续由父页向外暴露，runtime 只消费父页提供的 emit 回调。
