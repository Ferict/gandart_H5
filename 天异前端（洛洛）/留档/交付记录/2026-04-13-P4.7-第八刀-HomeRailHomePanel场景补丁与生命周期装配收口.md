## 2. 收口结果

1. `useHomeRailHomePanelRuntime` 不再直接持有以下装配顺序：
   - `useHomeScenePatchController(...)`
   - `useHomeRailHomeMarketReload(...)`
   - `useHomeRailHomeContentLifecycle(...)`
2. 新 `useHomeRailHomeSceneLifecycleRuntime` 统一承接：
   - 场景补丁 live apply
   - market reload / retry 桥接
   - 首屏初始化、refresh、activation check、visible-update check
3. `useHomeRailHomePanelRuntime` 当前只保留：
   - market query / remote-list pipeline
   - market result window
   - visual reveal + refresh presentation runtime
   - presentation state
   - effects

## 3. 风险控制

1. 本刀继续只搬装配层，不修改 scene patch、reload、content lifecycle 三个现有组合器的对外接口。
2. 新 helper 的输入类型直接复用既有组合器参数签名，避免在装配层重新发明口径。
3. `HomeRailHomePanel` 的对外可见接口未变化，仍保持 `refreshContent()` 与 `waitForRefreshPresentation()` 暴露路径不变。
