# 2026-04-13 G0.5 旧历史文件分类与回链字段规则

> 文档类型：执行记录  
> 状态：active  
> 更新时间：2026-04-13  
> 说明：本记录用于固定旧历史文件在迁移到新历史记录体系前的分类口径，以及迁入时必须补齐的最小回链字段。

## 1. 分类目标

本轮不是把旧历史目录整体复制到新目录，而是先把旧文件分成两类：

1. 应迁入新历史体系
2. 不应迁入，只保留冻结快照或最终进垃圾桶

## 2. A 类：应迁入新历史体系

满足以下任一条件的旧历史文件，应迁入 `留档/历史记录/`：

1. 对关键行为有事实记录价值
2. 对关键批次有追溯价值
3. 对后续审查、复盘或责任回链有证据价值
4. 能明确回答“何时、谁、做了什么、为什么做”

典型例子：

1. 按日期记录的历史对话正文
2. 已结束批次的正式审查与正式计划历史
3. 对关键迁移、结构治理、契约调整有证据价值的历史执行记录
4. 能帮助解释历史目录和旧批次关系的总账或索引

## 3. B 类：不迁入新历史体系

满足以下任一条件的旧历史文件，不应迁入新历史体系：

1. 纯重复镜像
2. 只剩失效入口作用
3. 大量引用已死路径且无独立历史价值
4. 纯模板壳、空壳文件、过程噪音
5. 迁入后只会增加历史目录噪音

典型例子：

1. 只起旧入口跳转作用的说明文件
2. 已被更正式记录完整覆盖的重复摘要
3. 仅为临时流程服务、没有长期追溯价值的空白模板

## 4. 迁入新历史体系时的最小字段

迁入新历史体系的历史文件，建议至少补齐以下元信息：

1. 原始路径
2. 原始类别
3. 对应批次或日期范围
4. 迁移批次
5. 迁移原因
6. 关联正式记录

## 5. 当前结论

当前结论固定为：

1. 新历史体系不是旧台账群的原样搬家。
2. 迁移前先分类，是后续低风险整理的必要前提。
3. 只有 A 类内容才值得进入新历史体系。
4. B 类内容后续可以在保留冻结快照后再进入垃圾桶。
