# 历史记录

> 文档类型：目录入口  
> 状态：active  
> 更新时间：2026-04-13  
> 说明：本目录是新历史记录体系的正式骨架，目标固定为让关键行为可追溯、可反查。

## 1. 目录定位

`留档/历史记录/` 负责承载：

1. 关键行为流水
2. 关键批次追溯
3. 可回查的历史事件链

它不是：

1. 现行规则正文目录
2. 垃圾桶
3. 旧台账的直接替代壳

## 3. 后续职责

后续重建完成后，本目录下的正式历史记录至少应能回答：

1. 何时
2. 谁
3. 做了什么
4. 为什么做
5. 依据是什么
6. 结果是什么
7. 回链到哪里

建议长期子目录为：

1. `对话/`
2. `批次档案/`
3. `索引与总账/`

建议迁移顺序为：

1. `项目索引/`
2. `对话台账/`
3. `审核与修改计划/`
4. `项目台账.md`

## 4. 使用边界

1. 新历史记录必须遵守 [07-文档与留档规则.md](/H:/工作/天异/uniapp+vue新架构/规则体系/07-文档与留档规则.md)。
2. 历史记录应回链到正式规则、正式计划、正式审查或正式执行记录。
3. 历史记录不得回链垃圾桶。
4. 旧历史根入口被冻结后，只承担回链和反查职责，不再承接现行写入。
