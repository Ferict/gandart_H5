# 审核 - 单位体系细则与项目骨架规则落地前审查

## 1. 审核背景

- 用户要求先落 `单位体系细则` 和 `正式项目骨架规则`。
- 同时要求启动 `UI 视觉专项优化` 与 `环境部署计划`。
- 用户明确要求：所有待进行、未确认、需要用户决策的事项，一律记台账，不得自行拍板。

## 2. 审核结论

1. 当前主规范已经允许进入细则阶段。
2. `单位体系细则` 和 `正式项目骨架规则` 属于现阶段主缺口，可以直接落正文。
3. `UI 视觉专项优化` 与 `环境部署` 仍包含多个待决事项，这一批只能开专题、立边界、列待决，不应直接落实技术方案。

## 3. 本批允许落地

- 新增两份规则正文：
  - `开发规范/单位体系细则.md`
  - `开发规范/正式项目骨架规则.md`
- 回改规范入口和主规范引用关系
- 新增专项审查/计划/索引文档
- 台账与对话台账补录

## 4. 本批禁止落地

- 不创建正式源码骨架
- 不安装依赖
- 不初始化 uni-app 运行环境
- 不决定包管理器、JS/TS、HBuilderX/CLI 主链
- 不决定第三方 UI 库或 lint 工具链

## 5. 待用户决策事项

1. `JS` 还是 `TS`
2. `npm / pnpm / yarn`
3. HBuilderX 为主还是 CLI 为主
4. 是否首批引入 Pinia 真实依赖
5. 是否首批引入第三方 UI 组件库
6. 是否首批引入 ESLint / Prettier / Stylelint
7. 环境部署优先支持哪些目标：
   - H5
   - App-Plus Android
   - App-Plus iOS
8. 客户端壳工程接入与打包由谁承接、何时介入
