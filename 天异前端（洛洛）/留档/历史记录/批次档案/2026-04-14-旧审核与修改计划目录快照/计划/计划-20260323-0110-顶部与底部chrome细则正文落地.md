# 计划：顶部与底部 chrome 细则正文落地

## 1. 批次目标

1. 建立 `开发规范/顶部与底部 chrome 细则.md`。
2. 把新细则接回开发规范入口与主规范。
3. 让“细则层规范落地总计划”正式进入阶段 A 的第 1 项已完成状态。
4. 同步更新当前审核摘要、提交清单、项目索引、项目台账与对话台账。

## 2. 实施步骤

1. 编写 chrome 细则正文。
2. 回改：
   - `开发规范/README.md`
   - `开发规范/前端开发规范整理版.md`
3. 更新：
   - `项目索引/细则层规范落地总计划.md`
   - `审核与修改计划/当前审核与修改计划.md`
   - `提交清单/当前提交清单.md`
   - `项目索引/00-总索引.md`
   - `项目台账.md`
   - `对话台账/对话台账-20260322.md`
4. 复查文档引用与边界。
5. Git 提交。

## 3. 风险控制

1. 不擅自把“顶部 chrome 细则”扩写成“视口与键盘细则”。
2. 不修改任何源码文件，避免把文档批次混成工程批次。
3. 不替后续页面实现拍板不属于本细则的视觉细节。

## 4. 验证方式

1. 检查新细则已被主入口和主规范引用。
2. 检查当前计划、提交清单、台账是否和本批次主题一致。
3. 检查 Git 工作区只包含本批次文档修改。
