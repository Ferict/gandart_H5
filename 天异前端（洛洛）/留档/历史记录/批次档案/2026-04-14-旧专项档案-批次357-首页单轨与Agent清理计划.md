## 一、目标

本批只做两件事：

1. 收掉首页壳层最后一层“设备能力分叉”，让现行首页真正只按宽度调整尺寸。
2. 清理 `Ask/Explore/Plan.agent.md` 中当前环境不识别的工具与模型，消除编辑器持续告警。

本批不做：

- 不改首页业务内容链
- 不动图片链与个人中心资产链
- 不启用 `768` 网页独立 UI
- 不动项目源码以外的其他 VS Code 全局设置

## 二、输入结论

### 2.1 首页壳层现行问题

当前首页虽然已经移除 `double-page / triple-page / visiblePages`，但仍存在以下残留：

1. `HomeShellTrackStage.vue` 仍根据 `pointerType / hoverCapable` 派生 `isMobileFloatingNav`
2. `HomeShellNavRail.vue` 仍保留 `floating` prop 与 `.is-floating` 浮动样式分支
3. `useResponsiveRailLayout.ts` 仍把 `pointerType / hoverCapable` 作为关键变更源
4. `homeShellLayoutMode.service.ts` 的 `ViewportRuntimeContext` 仍暴露 `pointerType / hoverCapable`

这会导致同样是 `>= 522` 宽度时，触控端与非触控端出现两套窄侧栏行为。

### 2.2 Copilot agent 现行问题

当前 3 个 agent 文件存在无效前言配置：

- `Ask.agent.md`
- `Explore.agent.md`
- `Plan.agent.md`

已知无效项：

- `github/issue_read`
- `github.vscode-pull-request-github/issue_fetch`
- `github.vscode-pull-request-github/activePullRequest`
- `Gemini 3 Flash (Preview) (copilot)`

根因：

1. 当前环境未安装 `GitHub Pull Requests and Issues`
2. 当前 Copilot Chat 环境不识别该 Gemini 模型名

## 三、计划复核结论

### 3.1 总体判断

本计划总体合理，可以执行。

原因：

1. `357A` 命中的是首页壳层最后一层真实分叉，属于根因修复，不是样式补丁。
2. `357B` 命中的是本机环境中已明确无效的前言配置，清理后不会影响项目源码运行。

### 3.2 必须写清的修订点

1. `522` 是设计阈值，不是当前运行时的立即触发点。
2. 由于现有 `HOME_LAYOUT_SWITCH_HYSTERESIS = 12` 仍保留，实际运行边界应写清：
   - 从 `single-page` 放大进入 `single-page-with-nav`：`534`
   - 从 `single-page-with-nav` 缩回 `single-page`：`510`
3. `768` 目前是冻结阈值，不参与现行运行逻辑；本批不能把它重新接回模式判定。
4. `.agent.md` 位于 VS Code `globalStorage`，不属于项目源码；即使清掉，本地扩展更新或重建 agent 后仍可能复发。

## 四、关键风险

### 4.1 风险一：把“设计阈值 522”误写成“运行时立即触发 522”

现状证据：

- `homeShellLayoutMode.service.ts` 中 `HOME_LAYOUT_SWITCH_HYSTERESIS = 12`
- 当前逻辑：
  - 前态为 `single-page` 时，需满足 `viewportWidth >= 522 + 12`
  - 前态为 `single-page-with-nav` 时，需满足 `viewportWidth < 522 - 12`

这意味着：

- 进入侧栏：`534`
- 退出侧栏：`510`

本批建议：

- 保留现有滞回，不在 `357A` 动它
- 计划与验收文档中同时写：
  - `522` 是设计阈值
  - `534 / 510` 是现行运行边界

### 4.2 风险二：`ViewportRuntimeContext` 删字段时把安全区消费者一起打坏

已核对当前消费面：

- `HomeShellDrawer.vue`
- `HomeShellNavRail.vue`
- `HomeShellTabbar.vue`
- `HomeShellTrackStage.vue`

其中真正消费 `pointerType / hoverCapable` 的只有：

- `HomeShellTrackStage.vue`
- `useResponsiveRailLayout.ts`
- `homeShellLayoutMode.service.ts`

其它组件只用：

- `safeAreaTop`
- `safeAreaBottom`
- `viewportWidth`
- `viewportHeight`

本批建议：

- 可以删 `pointerType / hoverCapable`
- 但只删这两个字段，不碰 safe-area 相关字段和 `resolveHomeShellInsets()`

### 4.3 风险三：把 `768` 常量误当成现行行为重新接回

已核对：

- `HOME_LAYOUT_MIN_WEB_STAGE_WIDTH = 768` 目前只剩记录用途
- 当前模式判定函数 `resolveHomeShellLayoutMode()` 实际只按：
  - `< 522`
  - `>= 522`

本批建议：

- 保留 `HOME_LAYOUT_MIN_WEB_STAGE_WIDTH = 768`
- 只保留注释说明它是后续网页 UI 的冻结阈值
- 本批禁止恢复任何：
  - `HOME_LAYOUT_MODE_WEB_STAGE`
  - `mode-web`
  - `>=768` 运行态特判

### 4.4 风险四：`.agent.md` 清理结果不是项目持久资产

这批改动文件位于：

- `C:\Users\1\AppData\Roaming\Code\User\globalStorage\github.copilot-chat\...`

这意味着：

1. 不会进入 `uniapp+vue 新架构` 源码本身的交付物
2. 可能被扩展更新覆盖
3. 可能被 Copilot Chat 重新生成 agent 文件覆盖

本批建议：

- 承认它是“本机环境清理”
- 不把这条写成“项目源码长期收口”
- 仅在项目计划文档中登记“本机需同步清理”

## 五、批次拆分

### 357A：首页壳层纯宽度单轨收口

#### 357A-1 改模型与服务边界

文件：

- `src/services/homeShellLayoutMode.service.ts`
- `src/composables/useResponsiveRailLayout.ts`

修改目标：

1. `ViewportRuntimeContext` 删除：
   - `pointerType`
   - `hoverCapable`
2. `DevicePointerType` 删除
3. `DeviceCapabilityFlags` 删除
4. `resolveDevicePointerType()` 整个函数删除
5. `resolveHomeShellViewportSnapshot()` 不再向 `runtimeContext` 塞入设备指针能力
6. `hasRuntimeContextChanged()` 删除对：
   - `current.pointerType !== next.pointerType`
   - `current.hoverCapable !== next.hoverCapable`
     的比较
7. `runtimeContext` 默认值删掉：
   - `pointerType: 'unknown'`
   - `hoverCapable: false`

落地代码口径：

```ts
export interface ViewportRuntimeContext {
  viewportWidth: number
  viewportHeight: number
  safeAreaTop: number
  safeAreaBottom: number
  hasVisualViewport: boolean
}
```

```ts
const hasRuntimeContextChanged = (
  current: ViewportRuntimeContext,
  next: ViewportRuntimeContext
): boolean => {
  return (
    current.viewportWidth !== next.viewportWidth ||
    current.viewportHeight !== next.viewportHeight ||
    current.safeAreaTop !== next.safeAreaTop ||
    current.safeAreaBottom !== next.safeAreaBottom ||
    current.hasVisualViewport !== next.hasVisualViewport
  )
}
```

8. `534`：从前态单页放大时进入 `single-page-with-nav`
9. `767`：仍是页面 `430 + 16 + 76`
10. `768`：当前仍保持 `single-page-with-nav`，不启用网页独立 UI
