# 批次367-活动页深度审查

## 审查前提

- 首页按当前用户边界已完成全面校对。
- 本轮开始从首页正式资源接口与页面链路口径出发，深审活动页。

## 审查结论

### 问题1：活动资源链没有真实视觉字段，详情链也不会消费图片

- `ContentActivityEntryDto` 仅含 `title / eyebrow / description / tone / badgeLabel / target`，没有 `asset` 或 `placeholderIconKey` 字段。
- `buildActivitySceneBlocks()` 仅把文字字段映射到 `activity_entries`。
- `buildActivityResource()` 将 `asset` 固定写成 `null`。
- `mapActivityResource()` 将 `imageUrl` 固定写成空字符串，并固定 `placeholderIconKey: 'aperture'`。
- 结果：
  - 活动页入口卡无法像首页卡片那样接图片资源。
  - 活动详情页即便后端未来返回 `resource.asset`，当前 mapper 也不会显示图片。

### 问题2：活动入口的 `badgeLabel` 端到端存在，但页面未消费

- `activitySceneDb.entries` 给 `asset-merge`、`priority-draw` 下发了 `badgeLabel: 'NEW'`。
- `content.mock.ts`、`homeRailActivityContent.service.ts` 已把它带入 `badgeLabel / badgeText`。
- `HomeRailActivityPanel.vue` 模板未渲染 `badgeText`。
- 结果：数据链存在，页面层丢弃，活动入口当前丧失状态角标表达。

### 风险项：活动公告标签与公告类型分离维护，后续容易漂移

- `activitySceneDb.notices.tags` 单独维护标签数组。
- `filteredNotices` 用 `notice.type === tag` 做严格匹配。
- 只要标签数组和 notice type 不一致，就会出现空标签页或漏数据。

## 当前不视为问题

- 活动页与公告页已经统一走 `navigateByUrlSafely()`。
- 活动页 `try/catch/finally` 与 shell fallback 已存在。

## 下一步建议

1. 先补活动资源接口：给 `ContentActivityEntryDto` 与活动资源详情链补 `asset / placeholderIconKey`。
2. 再补活动入口角标渲染。
3. 最后决定标签是否改成从 notices 动态派生，或继续手工维护但补校验。
