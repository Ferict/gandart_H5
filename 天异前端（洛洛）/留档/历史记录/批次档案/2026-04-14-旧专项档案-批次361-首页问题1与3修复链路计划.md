## 五、实际修复链路

### 361A：首页图片资源链闭环

#### 目标

让首页首发卡与市场卡在现有资源前提下，优先显示真实图片，不再稳定落占位态。

#### 实际做法

1. 在 `src/mocks/content-db/assets.ts` 新增首页卡片专用资源 ID，而不是直接复用 banner 语义 ID。

- 建议新增：
  - `ASSET-HOME-FEATURED-AX99`
  - `ASSET-HOME-MARKET-C01`
  - `ASSET-HOME-MARKET-C02`
  - `ASSET-HOME-MARKET-C03`
  - `ASSET-HOME-MARKET-C04`

2. 新资源记录先走“现有素材闭环”方案，不等待新美术切图。

- 每条记录都显式写 `variants.card`
- 如果当前仓里只有 banner 素材，则先把 `variants.card` 指向现有可用 jpg
- `originalUrl` 与 `variants.card` 保持一致
- `focalPoint` 保持 `{ x: 0.5, y: 0.5 }` 或按现有素材中心值写入

示例写法：

```ts
'ASSET-HOME-FEATURED-AX99': {
  assetId: 'ASSET-HOME-FEATURED-AX99',
  originalUrl: `${bannerBase}/ax99-banner.jpg`,
  width: 686,
  height: 336,
  focalPoint: { x: 0.5, y: 0.5 },
  variants: {
    card: `${bannerBase}/ax99-banner.jpg`,
    detail: `${bannerBase}/ax99-banner.jpg`,
  },
}
```

3. 在 `src/mocks/content-db/drops.ts` 把 `AX-99` 的 `assetId` 从 `null` 改成首页首发专用资源 ID。

```ts
assetId: 'ASSET-HOME-FEATURED-AX99'
```

4. 在 `src/mocks/content-db/market-items.ts` 把 `C-01 ~ C-04` 的 `assetId` 全部从 `null` 改成各自首页市场卡专用资源 ID。

```ts
assetId: 'ASSET-HOME-MARKET-C01'
```

5. 不改 `homeRailHomeContent.service.ts` 的主映射逻辑，除非复查发现 `resolveAssetUrl(asset, 'card')` 没命中 `variants.card`。

- 当前代码已经正确：
  - `card -> variants.card ?? originalUrl`
- 所以本批应该优先改数据，不动服务逻辑。

#### 这样做的理由

- 根因在内容源，不在页面层。
- 给首页卡片单独资源 ID，后续换图只改资源库，不再污染 banner 资源语义。
- 先闭环“有图可出”，后续若有正式卡图，再替换 `variants.card` 即可。

#### 必须避免的错误

- 不要直接把 `HomeRailHomePanel.vue` 改成强行显示占位图或写死图片 URL。
- 不要把 banner 资源 ID 直接塞给市场卡，导致首页语义混乱。
- 不要为了闭环图片链，顺手改资源详情页或内容服务契约。

### 361B：首页主面板安全导航收口

#### 目标

让首页三个主面板统一复用 `navigateByUrlSafely()`，收掉快速连点重复入栈风险。

#### 实际做法

1. `HomeRailHomePanel.vue`

- 删除本地：

```ts
const safeNavigate = (url: string) => {
  uni.navigateTo({ url })
}
```

- 改为从 `homeRailNavigation.service.ts` 直接引入：

```ts
import { ..., navigateByUrlSafely } from '../../../services/homeRailNavigation.service'
```

- 所有跳转调用统一改成：

```ts
navigateByUrlSafely(url)
```

- 需要替换的处理函数：
  - `handleBannerClick`
  - `handleAnnouncementClick`
  - `handleCollectionQuickEntryClick`
  - `handleMarketSearchClick`
  - `handleMarketDocumentClick`
  - `handleCollectionClick`

2. `HomeRailActivityPanel.vue`

- 增加引入：

```ts
import {
  buildActivityNoticeDetailUrl,
  buildContentResourceUrl,
  navigateByUrlSafely,
} from '../../../services/homeRailNavigation.service'
```

- 替换：

```ts
uni.navigateTo({ url: buildContentResourceUrl(...) })
```

为：

```ts
navigateByUrlSafely(buildContentResourceUrl(...))
```

- 替换：

```ts
uni.navigateTo({ url: buildActivityNoticeDetailUrl(notice) })
```

为：

```ts
navigateByUrlSafely(buildActivityNoticeDetailUrl(notice))
```

3. `HomeRailProfilePanel.vue`

- 删除本地：

```ts
const safeNavigate = (url: string) => {
  uni.navigateTo({ url })
}
```

- 增加引入：

```ts
import { ..., navigateByUrlSafely } from '../../../services/homeRailNavigation.service'
```

- 把以下函数统一切到 helper：
  - `handleOpenCommunity`
  - `handleOpenSettings`
  - `handleShowQr`
  - `handleQuickEntryClick`
  - `handleSearchAssets`
  - `handleAssetClick`
- `handleCopyAddress` 不动，因为它不是导航动作。

4. 不改 `navigateByUrlSafely()` 本体，除非首页联调后确认 helper 本身存在误判。

- 当前 helper 已有：
  - URL normalize
  - 同 URL `320ms` 重复保护
  - 当前页路径短路
- 本批应优先统一消费，不扩 helper 语义。

#### 这样做的理由

- 问题不是“没有 helper”，而是首页主面板没接 helper。
- 先统一消费层，比继续改 helper 更低风险。
- 这样能保证首页导航口径与已收口的 NavRail 一致。

#### 必须避免的错误

- 不要顺手把设置页、服务页、独立页的 `navigateTo` 一并改掉。
- 不要在页面层再写第二套去抖逻辑。
- 不要把复制、筛选、局部高亮这类非导航动作也强塞给 helper。
