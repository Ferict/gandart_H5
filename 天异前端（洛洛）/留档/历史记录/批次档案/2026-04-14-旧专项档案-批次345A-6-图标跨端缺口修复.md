# 批次345A-6-图标跨端缺口修复

## 批次目标

收口图标适配层在交付级审查中命中的 3 个缺口：

1. 将 APP 图标分支从 `APP-PLUS` 收紧为 `APP`，覆盖 Android / iOS / 鸿蒙 APP 侧。
2. 将字库 `font-display` 从 `block` 调整为 `swap`，降低壳层关键图标冷启动空白风险。
3. 为 `AetherIcon` 的 APP 字体图标分支补最小参数语义近似，使 `strokeWidth` 不再被完全静默忽略。

## 实际修改

- `src/components/AetherIcon.vue`
- `src/utils/aetherIcon.registry.ts`
- `src/static/iconfont/aether-iconfont.css`

## 修复结果

- `AetherIcon` 的 APP 图标分支已统一改为 `#ifdef APP`。
- `aetherIcon.registry.ts` 的 APP 字库映射与 getter 已统一改为 `#ifdef APP / #ifndef APP`。
- `@font-face` 已改为 `font-display: swap`，并补充字体平滑策略。
- APP 字体图标现在会根据 `strokeWidth` 计算轻量级缩放值，作为跨端视觉近似策略。

## 复查结论

- `APP-PLUS` 专属宏边界问题：已收口。
- `font-display: block` 首屏空白风险：已收口。
- `AetherIcon` APP 分支完全忽略 `strokeWidth`：已收口为近似语义，不再是静默丢参。

## 验证结果

- `npm run type-check` 通过
- `npm run build:h5` 通过
- `npm exec uni build -- --platform app` 通过

## 剩余事项

- 这轮修复的是跨端代码路径与构建交付缺口，不是最终真机验收结论。
