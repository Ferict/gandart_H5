# 批次352 首页壳层宽度口径回收与768冻结

## 目标

- 按现行正确口径收回首页壳层宽度与模式逻辑。
- 当前运行态只保留 `single-page` 与 `single-page-with-nav`。
- `768` 仅登记为待启用网页阈值，网页端 UI 完成前不进入单独运行分支。
- 彻底移除三页并排与折叠屏兼容残留。

## 固定数值

- 基准宽：`375`
- 页面最大宽：`430`
- 窄侧边栏宽：`76`
- 页面与窄侧边栏间距：`16`
- 窄侧边栏进入阈值：`522`
- 待启用网页阈值：`768`
- 节流：`requestAnimationFrame + minIntervalMs = 80`
- 滞回：`12`

## 当前行为

1. `< 375`

- `single-page`
- 全局缩放，按 `viewportWidth / 375`

2. `375 - 521`

- `single-page`
- 页面内容不再继续拉宽超过 `430`

3. `>= 522`

- `single-page-with-nav`
- 页面 `430 + 间距 16 + 窄侧边栏 76`

4. `>= 768`

- 现行运行态仍保持 `single-page-with-nav`
- 不进入单独网页布局分支
- `768` 只作为后续网页 UI 的待启用阈值记录
