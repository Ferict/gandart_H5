# P11.12.5 backend-http 接轨风险与脏链隔离审查

> 文档类型：审查记录
> 状态：已修复
> 更新日期：2026-04-25
> 关联计划：`留档/计划记录/2026-04-25-P11.12.5-审查问题修复与脏链隔离计划.md`

## 审查结论

本批审查确认 `backend-http` 真实接驳存在四类需要立即修复的问题，且当前工作树同时存在二线 UI 脏链与未跟踪来源包。本批只处理接驳风险与文档门禁，不回滚、不提交二线 UI，不删除任何文件。

## 问题清单

1. `home/activity/profile` 场景聚合只用主响应 envelope 生成成功结果。任一必需 wrapper 返回 `code != 0` 时，页面仍可能拿到成功 scene，属于真实接驳风险。
2. 首页首发为空时会调用空对象归一化，生成 `drop_1 / 首发藏品 / 0` 的假首发块，容易把缺数据伪装成业务数据。
3. 市场列表没有确认旧后端支持排序参数；当前若直接下发猜测字段，会形成未确认接口依赖。
4. `VITE_CONTENT_PROVIDER` 未知值会静默落到 mock，真实联调时可能误以为已经走后端。
5. `留档/审查记录/2026-04-25-P11.12.3A0-最新版后端契约迁入审查.md` 未被格式门禁接受，需要恢复 `lint:format`。
6. `SecondaryPageTopbar.vue`、`HomeRailTopbar.vue`、`src/pages/updating/**` 和 `来源源码/` 属于二线或来源脏链，本批不得提交。

## 处理裁定

1. 场景必需 wrapper 采用严格失败策略：任一业务失败，整个 scene 返回失败 envelope。
2. 空首发只允许输出空壳，不允许生成假 ID、假标题或假业务内容。
3. 市场排序本批只在 `backend-http` adapter 对当前页返回结果做稳定排序；跨页全局排序进入“需要后端补充口径”。
4. provider 只允许 `mock`、`http`、`backend-http`，未知值直接抛错。
5. 二线 UI 改动只记录越界风险，不在本批处理。
