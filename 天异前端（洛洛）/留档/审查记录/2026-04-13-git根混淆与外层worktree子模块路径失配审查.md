# 2026-04-13 Git 根混淆与外层 worktree 子模块路径失配审查

> 说明：本记录用于留存当前会话环境下 `git` 根混淆的诊断结果，明确实际应用仓、外层 worktree 壳仓，以及导致 `git` 判断失真的直接原因。

## 1. 结论

1. 当前应用工作并非“没有 git”，而是存在两层 `git`：
   - 外层会话壳仓：`C:\Users\1\.codex\worktrees\55dc\天异`
   - 实际应用仓：`H:\工作\天异\uniapp+vue新架构`
2. 对本应用来说，唯一有效的 `git` 根应是：
   - [H:\工作\天异\uniapp+vue新架构](/H:/工作/天异/uniapp+vue新架构/README.md)
3. 外层壳仓之所以表现异常，不是应用仓损坏，而是外层对子模块路径的记录失配。

## 2. 诊断结果

1. 外层壳仓 `.gitmodules` 当前配置的是：
   - `uniapp+vue新架构`
2. 但外层壳仓历史索引里仍残留旧 gitlink 路径：
   - `uniapp+vue 新架构`
3. 这导致外层仓会同时看到：
   - 旧路径删除
   - 新路径新增
4. 并直接触发：
   - `git submodule status` 报错 `fatal: no submodule mapping found in .gitmodules for path 'uniapp+vue 新架构'`

## 3. 影响

1. 在外层壳仓查看 `git status`，会误以为应用目录整体被删除并重加。
2. 在外层壳仓执行子模块相关命令，会因为旧路径失配直接失败。
3. 应用仓本体 [H:\工作\天异\uniapp+vue新架构](/H:/工作/天异/uniapp+vue新架构/README.md) 的 `git` 本身正常，但会被外层壳仓的异常表现干扰判断。

## 4. 当前口径

1. 本应用所有源码、测试、规则、留档相关 `git` 判断，应只在：
   - [H:\工作\天异\uniapp+vue新架构](/H:/工作/天异/uniapp+vue新架构/README.md)
     执行。
2. 外层壳仓只作为当前会话承载环境，不作为本应用的真实提交边界依据。
