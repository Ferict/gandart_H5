# 2026-04-13 P4.1 tests/unit 结构热区审查

> 文档类型：审查记录
> 状态：active
> 更新时间：2026-04-13
> 说明：本审查用于判断 `tests/unit/` 是否已形成需要继续治理的平铺热区。

## 1. 审查范围

本次只审查：

1. `tests/unit/` 当前文件集。
2. 单测命名簇与来源目录的关系。
3. 是否具备低风险分组条件。

## 2. 当前现状

当前 `tests/unit/` 有 `41` 个单测文件，全部平铺在一层。

这些文件已经形成明显职责簇：

1. `content*`
2. `home*` / `useHome*` / `useActivity*` / `useProfile*`
3. `profileAssetDetail*` / `useProfileAssetDetail*`
4. `routeQuery / safeLogger / noticeTime` 等共享工具类

## 3. 审查结论

`tests/unit/` 当前属于值得继续治理的结构热区。

原因是：

1. 文件数量已高于人工平铺检索的舒适范围。
2. 单测职责簇稳定。
3. 大部分测试通过别名导入源码，物理迁移的相对风险较低。

## 4. 建议

下一批建议按以下分组：

1. `content/`
2. `home/`
3. `profile-asset-detail/`
4. `shared/`

这样可以把单测目录结构和当前已治理过的源码主簇对齐。
