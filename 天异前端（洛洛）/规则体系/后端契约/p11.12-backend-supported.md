## 3. 当前支持层级

当前迁入后的后端契约支持层级固定为：

1. `confirmed`
   - 可作为 P11.12 后续适配与接线的稳定候选依据。
2. `exported-unused`
   - 已抽出 wrapper，但未证明当前前端稳定使用；只能作为候选，不能直接接 UI。
3. `unresolved`
   - 仍待确认、待补证、待后端签收。
4. `inferred`
   - 推断层，只作线索，不可直接当后端承诺。

## 5. 当前已见字段证据

以下只表示“证据层已看到”，不表示当前前端一定要展示：

| 字段或语义   | 证据状态                  | 当前解释                                                                                       |
| ------------ | ------------------------- | ---------------------------------------------------------------------------------------------- |
| `total_num`  | 旧前端多处消费            | 旧前端用于“限量/总量”展示，可作为当前市场卡和详情卡总量/限量候选来源。                         |
| `limited`    | 旧前端消费                | 旧前端用于“限量”展示，可作为商城或类似藏品总量候选来源。                                       |
| `flux`       | 旧前端消费                | 旧前端用于“流通”展示；当前 UI 未确认要展示，不能替代“总量/限量”。                              |
| 用户持有数量 | 有旧前端个人藏品接口线索  | 可作为个人藏品卡“单个用户持有量”的候选来源，但需要在 P11.12.3A 盘点时确认具体 wrapper 和字段。 |
| 入库时间     | 有旧前端资产/藏品详情线索 | 可作为详情页“入库时间”候选来源，但需要确认具体字段名、格式和时区。                             |
| 版本         | 有旧前端藏品详情展示需求  | 可作为详情页“版本”候选来源，但需要确认字段名、枚举和展示文本。                                 |

## 6. 当前不算已支持的内容

以下内容不得从本文件推导为“后端已支持当前 UI”：

1. 当前 UI 没有任何位置需要展示“成交量”；即使后端或旧前端存在交易、成交、流通字段，也不自动进入当前展示需求。
2. 链上溯源字段不算已签收：`owner`、`contract`、`chain`、`tokenStandard`、`provenance`。
3. 多维属性字段不算已签收：`traits`。
4. 只出现在 `unresolved.md`、`contracts/inferred/*` 或缺真实响应样例的字段，不算已支持。
