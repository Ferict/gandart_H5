## 1. 文件定位

本文件是“质量工具运行说明”，不是业务源码，也不是正式规则真值源。

稳定规则以以下文件为准：

1. [规则体系/02-工程与工具规则.md](./规则体系/02-工程与工具规则.md)
2. [规则体系/03-安全与边界规则.md](./规则体系/03-安全与边界规则.md)

正式命令入口以 [package.json](./package.json) 的 `scripts` 为准。

本文件只回答：

1. 当前用了哪些质量工具。
2. 日常、本地、CI 分别跑哪些命令。
3. 哪些命令需要额外环境或系统工具。
4. 依赖审计、外链审计、AI 评测这类专项门禁当前怎么判断。

## 2. 质量工具是什么

质量工具是项目的自动质检线，用来在提交、CI 或发版前发现明显问题。

它们不实现业务功能，只负责检查：

1. 类型是否正确。
2. 代码和样式是否符合规则。
3. 测试是否通过。
4. 文档格式和基础措辞是否稳定。
5. 依赖和外链是否触碰安全边界。
6. 是否出现未使用文件、未登记依赖或仓规违规。

## 3. 当前命令分层

### 3.1 日常开发检查

适合普通代码改动后先跑：

```bash
npm run type-check
npm run lint
npm run test:unit
```

含义：

1. `type-check`：跑 Vue / TypeScript 类型检查。
2. `lint`：跑 ESLint、Stylelint、Prettier 格式检查。
3. `test:unit`：跑 Vitest 单元测试。

### 3.2 仓级质量检查

当前主质量入口：

```bash
npm run quality:check
```

它当前包含：

1. `npm run type-check`
2. `npm run lint`
3. `npm run test:unit`
4. `npm run knip`
5. `npm run lint:docs`

扩展质量入口：

```bash
npm run quality:full
```

它在 `quality:check` 基础上追加：

1. `npm run lint:semgrep`
2. `npm run lint:vale`

### 3.3 专项检查

专项命令：

```bash
npm run lint:external-resources
npm run audit:deps
npm run test:e2e
npm run check:content:http
npm run ai:eval:smoke
```

说明：

1. `lint:external-resources` 检查 H5 入口外部脚本和样式资源是否满足白名单与 SRI 口径。
2. `audit:deps` 检查生产依赖漏洞是否超过已登记安全基线。
3. `test:e2e` 跑 Playwright H5 烟测。
4. `check:content:http` 需要真实后端 base URL 和样本参数，当前不是默认无条件可跑的命令。
5. `ai:eval:smoke` 需要 `PROMPTFOO_PROVIDER` 和对应模型密钥，不能在缺环境时伪造通过结论。

## 4. 当前工具清单

### 4.1 代码、类型、样式

1. `vue-tsc`：Vue / TypeScript 类型检查。
2. `ESLint`：TypeScript / Vue 代码静态检查。
3. `Stylelint`：样式和 Vue 样式块检查。
4. `Prettier`：代码、配置、文档格式检查。

### 4.2 测试与冗余检查

1. `Vitest`：单元测试。
2. `Playwright`：H5 端到端烟测。
3. `Knip`：未使用文件、导出、依赖和二进制入口检查。

### 4.3 文档与仓规检查

1. `textlint`：文档基础规则检查。
2. `Semgrep`：安全和仓规类静态扫描。
3. `Vale`：文档术语和措辞检查。

### 4.4 提交流程

1. `Lefthook`：提交前和推送前执行本地门禁。
2. `Commitlint`：约束提交信息格式。
3. `Reviewdog`：在 GitHub PR 中回贴 ESLint 增量结果。

### 4.5 AI 辅助工具

1. `Promptfoo`：AI 规则 smoke 评测。
2. `Continue CLI`：本地 AI review 辅助。

AI 工具只作为辅助信号，不能替代类型检查、测试、lint 或人工审查。

## 5. 系统级工具准备

Windows 下可直接执行：

```powershell
npm run setup:quality:windows
```

该命令负责：

1. 安装 `Semgrep`。
2. 安装 `Vale`。
3. 下载 `Reviewdog` 到本地 `.tools/bin`。
4. 安装 `Playwright Chromium`。
5. 安装 `Lefthook` hooks。

## 6. CI 当前口径

GitHub Actions 当前主要入口是 [.github/workflows/quality-gate.yml](./.github/workflows/quality-gate.yml)。

当前 CI 会运行：

1. `npm ci`
2. `npm run type-check`
3. `npm run test:unit`
4. `npm run knip`
5. `npm run lint:docs`
6. `npm run lint:external-resources`
7. PR 场景下的 Reviewdog ESLint 增量回贴
8. `npm run lint`
9. `npm run lint:semgrep`
10. `npm run lint:vale`
11. 条件满足时的 `npm run ai:eval:smoke`
12. `npm run audit:deps`

`audit:deps` 当前不是纯展示命令：

1. 当前生产依赖接受基线为 `critical <= 0`、`high <= 18`。
2. 当前基线内通过。
3. 如果生产依赖出现 `critical > 0` 或 `high > 18`，命令返回非零退出并阻断 CI。
4. 依赖漏洞接受原因和升级边界见 [依赖与外链安全基线清单.md](./依赖与外链安全基线清单.md)。

## 7. 外链资源门禁

当前 H5 入口未接入外部脚本或样式 CDN 资源。

若未来新增第三方外部资源：

1. 第三方 `script` 必须登记白名单并补 `integrity` 与 `crossorigin="anonymous"`。
2. 第三方 `link rel="stylesheet"` 必须登记白名单并补 `integrity` 与 `crossorigin="anonymous"`。
3. `preload` / `modulepreload` 如果指向第三方可执行资源，也按同样规则处理。
4. 普通内容图片链接不属于本门禁的脚本/样式执行资源范围。

## 8. Promptfoo 环境变量

`Promptfoo` 不绑定固定模型，需要运行者设置 provider。

OpenAI 示例：

```powershell
$env:PROMPTFOO_PROVIDER = "openai:gpt-4.1-mini"
$env:OPENAI_API_KEY = "你的 key"
npm run ai:eval:smoke
```

GitHub Actions 中只在 PR 且存在 `PROMPTFOO_PROVIDER` 与 `OPENAI_API_KEY` 时运行 Promptfoo smoke eval。

workflow 中真实密钥只注入 Promptfoo step；step `if:` 只读取 job 级 `HAS_*` 标记，不直接判断 `secrets.*`。

## 9. GitHub 联调源码仓质量资产

以下质量资产随 GitHub 联调源码仓交付：

1. `promptfoo/`
2. `semgrep/`
3. `vale-styles/`
4. `.continue/`

以下文档和规则入口也随 GitHub 联调源码仓交付：

1. `AGENTS.md`
2. `规则体系/`
3. active `留档/`
4. `提交清单/README.md`
5. `垃圾桶/README.md`

原因：

1. `package.json`、CI workflow 和 `scripts/quality/*` 会直接读取这些目录。
2. `npm run lint:docs` 和 `npm run lint:format` 会直接检查这些文档和规则入口。
3. 如果交付仓缺这些目录，CI 和本地质量门禁会缺少规则、配置、文档或评测资产。

以下目录不随源码交付：

1. `.tools/`
2. `留档/历史记录/`

原因：

1. `.tools/` 是 Reviewdog 等工具的本地下载和生成目录，不是源码仓必需输入。
2. `留档/历史记录/` 当前不在 active 文档门禁覆盖范围内，不作为联调源码仓必需输入。
