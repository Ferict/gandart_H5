# 2026-04-13 P4.2 首页巨型组件拆分顺序与边界计划

> 文档类型：计划记录
> 状态：active
> 更新时间：2026-04-13
> 说明：本计划用于承接首页巨型组件结构审查结论，明确下一阶段的拆分顺序、每批边界和验证要求。

## 1. 目标

本阶段的目标固定为：

1. 降低首页三块 rail panel 的脚本组合层复杂度。
2. 让首页组件回到“页面装配组件 + 成组运行时 composable”的可维护结构。
3. 不再继续扩散更多零碎微文件。

## 2. 总策略

下一阶段采用以下默认策略：

1. 先拆脚本 orchestration，再考虑模板子组件。
2. 一次只处理一个 rail panel。
3. 每次优先抽出“一整组职责”，不抽零散单函数。
4. 每批都要先跑最小验证，再扩大范围。

## 3. 批次顺序

### 3.1 第一批

目标文件：

1. [HomeRailHomePanel.vue](/H:/工作/天异/uniapp+vue新架构/src/pages/home/components/HomeRailHomePanel.vue)

优先抽离方向：

1. 首页内容 shell 与 scene hydrate 编排
2. refresh / activation / visible update 协调
3. 点击导航与 target URL 组装

不优先处理：

1. banner / featured / market 子模板切组件
2. placeholder icon 等小计算的机械拆散

### 3.2 第二批

目标文件：

1. [HomeRailProfilePanel.vue](/H:/工作/天异/uniapp+vue新架构/src/pages/home/components/HomeRailProfilePanel.vue)

优先抽离方向：

1. profile scene + asset list 的页面级 orchestration
2. load-more / observer / visible asset 窗口协调
3. quick action 与详情跳转之外的运行时编排

### 3.3 第三批

目标文件：

1. [HomeRailActivityPanel.vue](/H:/工作/天异/uniapp+vue新架构/src/pages/home/components/HomeRailActivityPanel.vue)

优先抽离方向：

1. notice list reload / activation / visible update 协调
2. first-screen / bottom footer 状态编排

### 3.4 暂缓批次

暂不优先处理：

1. [HomeShellTrackStage.vue](/H:/工作/天异/uniapp+vue新架构/src/pages/home/components/HomeShellTrackStage.vue)

原因：

1. 当前职责相对集中
2. 不属于最痛的人工维护热点

## 4. 新增文件边界

后续新增 composable 时，执行以下边界：

1. 首页首页市场 rail 新增到 [src/pages/home/composables/home](/H:/工作/天异/uniapp+vue新架构/src/pages/home/composables/home/README.md)
2. 个人页 rail 新增到 [src/pages/home/composables/profile](/H:/工作/天异/uniapp+vue新架构/src/pages/home/composables/profile/README.md)
3. 活动页 rail 新增到 [src/pages/home/composables/activity](/H:/工作/天异/uniapp+vue新架构/src/pages/home/composables/activity/README.md)
4. 不新增新的 `src/pages/home/composables` 一级分组

## 5. 结构约束

后续拆分必须遵守：

1. 不把页面专属逻辑重新塞回 root `src/services/`
2. 不新增无业务归属的 `utils` 垃圾桶
3. 不把三个 rail panel 共同依赖的偶发细节强行抽成伪共享
4. 不为了降行数继续制造更多无意义微文件

## 6. 每批验证要求

每次组件拆分后，至少执行：

1. `git diff --check`
2. `npm run type-check`
3. `npm run knip`
4. `npm run test:unit`

如果批次触达运行时行为，再补：

1. `npm run test:e2e`
