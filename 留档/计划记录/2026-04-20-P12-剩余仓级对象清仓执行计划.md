## 1. 目标

1. 清理当前跟踪路径里仍引用旧规则入口的仓级工具配置。
2. 收口质量工具忽略配置中已退场旧路径的残留。
3. 修正 `垃圾桶/README.md` 的本仓隔离入口说明。
4. 统一三份根现行说明文档头部元信息。
5. 回写 README、提交清单、P12 主线计划和 exec 计划，使下一步能稳定进入 P13。

## 2. 范围

允许修改：

1. `.continue/checks/rule-compliance-review.md`
2. `promptfoo/promptfooconfig.yaml`
3. `.eslintrc.cjs`
4. `.prettierignore`
5. `.stylelintignore`
6. `.textlintignore`
7. `package.json`
8. `README.md`
9. `源码交付白名单.md`
10. `质量门禁工具接入说明.md`
11. `依赖与外链安全基线清单.md`
12. `提交清单/README.md`
13. `垃圾桶/README.md`
14. 本轮新增或回写的 `留档/审查记录/`、`留档/计划记录/`、`留档/交付记录/`

## 3. 不做什么

1. 不移动、删除或编辑 `src/` 活跃源码。
2. 不移动、删除或编辑 `tests/`。
3. 不改内容域正式 API 契约。
4. 不读取 `垃圾桶/` 中旧内容作为现行依据。
5. 不清理 `新藏品/` 的本地图片素材。
6. 不进入 `P13` 活跃源码兼容旁路线审查。

## 4. 执行顺序

1. 更新 `.continue` 与 `promptfoo` 中的旧规则入口断言。
2. 更新质量工具 ignore 配置，移除已退场旧路径忽略项，并保留 `垃圾桶/README.md` 可被格式和文档检查覆盖。
3. 更新 `package.json` 的 `lint:docs` 范围，让当前仓级工具目录 README、`提交清单/README.md` 和 `垃圾桶/README.md` 进入文档检查。
4. 重写 `垃圾桶/README.md` 为本仓项目内隔离入口说明。
5. 更新根 README、三份根现行说明和 `提交清单/README.md` 的边界表述。
6. 回填 P12 审查、计划、exec 计划和主线规划状态。
7. 写入 P12 交付记录。
8. 运行最小检查并提交。

## 5. 验收标准

1. 当前跟踪路径中不再由 `.continue` 或 `promptfoo` 主动要求旧规则入口文件。
2. 当前质量工具忽略配置不再保留 `开发规范/`、`项目索引/`、`审核与修改计划/`、`对话台账/` 等退役路径忽略项。
3. `references/README.md`、`提交清单/README.md`、`垃圾桶/README.md` 能被文档检查覆盖。
4. `垃圾桶/README.md` 明确说明本仓内 `垃圾桶/` 不是现行依据，且 Git 只保留 README。
5. P12 主线规划回填为已完成本轮仓级对象复核，下一步为 P13。
6. `git diff --check` 通过。
7. `npm run lint:docs` 通过。
8. `npx prettier --check` 覆盖本轮改动文件并通过。

## 6. 停止边界

1. 如果清理需要改动 `src/`、`tests/` 或真实接口契约，立即停止。
2. 如果发现需要新建长期目录，立即停止。
3. 如果发现 `新藏品/` 是否进入 Git 需要用户拍板，立即停止并只记录现状。
4. 如果文档检查暴露大量历史留档存量问题，停止并只记录检查结果，不扩题修历史。

## 7. 失败退出条件

1. 工作树出现非本轮改动。
2. 计划内对象与实际仓库结构不一致，导致继续执行会扩大范围。
3. `lint:docs` 或格式检查失败且失败点不属于本轮改动。
4. 需要用户决定目录去留但 exec 无法交互。

## 8. 完成后汇报要求

1. 回报已修正的仓级对象。
2. 回报保留对象和保留理由。
3. 回报 `新藏品/` 的本地-only 状态。
4. 回报检查命令和结果。
5. 回报交付记录路径和 Git 提交索引。
6. 明确下一步是否进入 P13。
