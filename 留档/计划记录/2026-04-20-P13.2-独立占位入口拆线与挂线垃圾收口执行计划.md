## 1. 目标

1. 拆掉七个独立占位页入口。
2. 保留现有导航函数名，统一返回 `updating` URL，降低调用方改动面。
3. 保留 `updating` 作为统一未完成功能承载页，通过 query 文案区分入口。
4. 把页面私有 query 和建设中文案收回页面附近。
5. 将退场源码转入用户确认的本地垃圾桶冻结区，不执行物理删除。
6. 移除 notice placeholder 内容链，不触碰正式公告 DTO/mock/http 链。

## 2. 范围

1. 修改 `src/services/home-rail/homeRailNavigation.service.ts`。
2. 修改 `src/pages.json`。
3. 修改 `src/pages/updating/` 和 `src/pages/profile-asset-detail/runtime/`。
4. 回写 `src/README.md` 及相关目录 README。
5. 转移七个独立占位页目录、`page-entry`、`route-query`、`pageContent` 旁路线。
6. 新增和回写 P13.2 正式留档。

## 3. 不做什么

1. 不新增真实后端接口。
2. 不修改 `src/contracts/content-api.contract.ts` 的正式接口结构。
3. 不移除正式内容域公告 DTO、mock、http 或 service。
4. 不处理首页内部过度拆分。
5. 不执行任何删除命令，不使用 `git rm`、`Remove-Item`、`rm`、`del`、`rmdir`、`git clean`。

## 4. 顺序

1. 导航函数统一改为生成 `/pages/updating/index?...`。
2. `updating` 本地化 query 解析和文案解析。
3. `profile-asset-detail` 本地化 query 解析。
4. `pages.json` 移除七个独立占位入口。
5. README 和正式留档回写现行口径。
6. 路径校验后把退场源码移动到 `垃圾桶/P13.2-挂线占位入口/`。
7. 运行检查并修复本批引入问题。
8. 形成独立 Git 提交。

## 5. 停止边界

1. 发现拆线会影响首页、活动、个人页或藏品详情的当前主流程。
2. 发现需要新增长期顶层目录或新的长期留档分类目录。
3. 发现正式内容域接口链被误伤。
4. 发现 `垃圾桶/` 落点与用户确认不一致。
5. 发现检查失败且原因超出本批范围。

## 6. 失败退出条件

1. 类型检查、单测、`knip`、lint 或构建失败且无法判定是否由本批引入。
2. Git 工作树出现非本批 dirty 文件。
3. 必须恢复旧独立占位页才能通过构建。
4. 必须修改正式后端接口契约才能继续。

## 7. 完成后汇报要求

1. 说明拆掉了哪些入口。
2. 说明保留了哪些入口和理由。
3. 说明转移到本地垃圾桶的对象。
4. 说明检查命令和结果。
5. 说明交付记录和 Git 提交索引。
6. 说明下一步是否进入 P16 或首页简化计划。

## 8. 验收标准

1. `pages.json` 只保留真实页面和统一 `updating` 页面。
2. 旧导航函数名仍导出，但不再返回旧独立占位页 URL。
3. 现行源码无 `page-entry`、`route-query`、`pageContent` 占位链引用。
4. 退场对象已转移到本地垃圾桶冻结区。
5. 门禁检查按交付记录完成。
