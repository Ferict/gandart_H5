# 2026-04-13 G0.6.1 剩余专题规则吸收计划

> 文档类型：详细计划
> 状态：active
> 更新时间：2026-04-13
> 说明：本计划用于收口当前仍阻塞 `G0.7` 的两块旧规则专题：UI 细颗粒专题，以及内容域运行时专题。

## 1. 目标

本轮目标固定为：

1. 把旧 `A09-A15` 中仍然有效的 UI 细颗粒口径迁入新规则体系。
2. 把旧 `A17-A19` 中仍然有效的内容域运行时口径迁入新规则体系。
3. 让新体系在这两块专题上不再依赖旧规则目录补充说明。

## 2. 范围

本计划只覆盖规则正文重建，不覆盖源码整理。

本轮不做：

1. 不改 `src/`
2. 不改 `tests/`
3. 不改 `backend-handoff/`
4. 不做 `G0.7` 物理迁移

## 3. 需吸收的旧规则对象

### 3.1 UI 细颗粒专题

1. `A09-顶部与底部Chrome法则.md`
2. `A10-视口与键盘法则.md`
3. `A11-热区与交互状态法则.md`
4. `A12-滚动容器法则.md`
5. `A13-小字号与文字容器法则.md`
6. `A14-毛玻璃与降级法则.md`
7. `A15-旧项目UI迁移重写法则.md`

### 3.2 内容域运行时专题

1. `A17-内容域前端接线与刷新更新法则.md`
2. `A18-内容卡片列表加载与图片反馈法则.md`
3. `A19-内容资源交付与缓存边界法则.md`

## 4. 新规则落点建议

为了保持目录可维护、但仍保留足够规则密度，建议新增两份专题正文：

1. `规则体系/08-交互与适配专题细则.md`
   - 吸收 `A09-A15`
2. `规则体系/09-内容域运行时与缓存专题细则.md`
   - 吸收 `A17-A19`

原因：

1. 继续硬塞进 `04-前端实现规则.md` 会让该文件重新巨石化。
2. UI 细颗粒专题彼此高度相关，适合合并为一份专题正文。
3. 内容域运行时、加载反馈、缓存边界也属于一个维护域，适合合并为一份专题正文。

## 5. 吸收原则

### 5.1 保留的内容

以下内容应保留为现行规则：

1. 仍对人工维护和实现边界有直接约束价值的条款
2. 已被当前实际代码与批次收口反复验证的口径
3. 用户已明确拍板、且后续仍会复用的规则

### 5.2 不保留的内容

以下内容不应机械迁入：

1. 只服务于旧规则体系内部引用链的条款编号
2. 只服务于旧台账结构的落账要求
3. 已被新规则体系上位原则替代的重复表述
4. 明显带有旧阶段流程噪音的说明性文字

## 6. 预期产物

本轮至少应产出：

1. `规则体系/08-交互与适配专题细则.md`
2. `规则体系/09-内容域运行时与缓存专题细则.md`

5. 更新 `规则体系/附录/02-废弃与冲突清单.md`
6. 一份正式执行记录

## 7. 验收标准

只有同时满足以下条件，才算这一轮完成：

1. `A09-A15` 不再阻塞 `G0.7`
2. `A17-A19` 不再阻塞 `G0.7`
3. 新专题正文能独立回答对应主题问题
4. 不再需要回读旧规则目录才能解释当前 UI 或内容域专题口径

3. 再进入 `G0.7`
