# 2026-04-13 P1 后端直连接入能力补齐详细计划

> 文档类型：详细计划
> 状态：active
> 更新时间：2026-04-13
> 说明：本计划用于把“已经留出 API”从契约和 adapter 的静态存在，升级为后端可直接接入、前端可直接切到 `http` provider、仓内可直接验证的可执行能力。

## 1. 目标

`P1` 的目标固定为三件事：

1. 给后端一个单一、明确、长期可维护的接入入口。
2. 给前端一个明确、可执行、可复用的 `http` provider 联调入口。
3. 给仓内一个明确的联调验证链，避免继续靠“契约 + mock + adapter”冒充已可接入。

## 2. 范围

本轮只处理内容域 5 个正式接口的直连接入能力，不扩新接口，不改正式 contract 范围：

1. `GET /api/content/scene`
2. `GET /api/content/resource`
3. `GET /api/content/list`
4. `POST /api/content/action/notice-read`
5. `POST /api/content/action/service-reminder-consume`

本轮不做：

1. 不改 `src/` 业务逻辑口径
2. 不改 `page + pageSize` 为 `cursor`
3. 不改后端字段范围
4. 不做 5 个接口的 readiness 逐条裁决，那是 `P2`

## 3. 预期产物

本轮至少产出：

1. `接口契约/` 顶层目录骨架

## 5. 配置与启动链补齐

### 5.1 前端配置入口

本轮应补齐：

1. `.env.example`
2. `http` provider 启动示例
3. `VITE_CONTENT_PROVIDER=http`

约束：

### 5.2 前端启动说明

## 6. 联调验证链

本轮必须建立一条仓内可执行的验证链，至少满足：

1. 能检查当前是否处于 `http` provider
2. 能检查 base URL 是否存在
3. 能对正式接口发起最小请求
4. 能输出明确成功 / 失败结论

推荐形式：

1. 脚本：
   - `scripts/check-content-http-link.ps1`
   - 或同等 Node 脚本
2. 文档：
   - [接口契约/内容域联调说明.md](/H:/工作/天异/uniapp+vue新架构/接口契约/内容域联调说明.md) 中记录执行方式与判定标准

约束：

1. 没有真实后端 URL 时，不虚构“已联通”
2. 没有验证链时，不允许宣称“后端可直接接入”
