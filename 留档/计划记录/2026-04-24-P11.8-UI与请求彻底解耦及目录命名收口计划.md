## 1. 当前定位

`P11.8` 不是废弃计划，而是 `P11.10` 的硬前置门槛。  
当前状态已经从“搁置待续”恢复并完成，后续只作为已达成门槛保留回链，不再单独扩写新的实施内容。

## 2. 本计划目标

`P11.8` 固定只做三件事：

1. 把 UI 展示链和请求链彻底拆开，不让页面和组件继续理解 transport 语义。
2. 解决 `src/adapters/` 空壳问题，让 adapter 成为真实的归一化层。
3. 把目录命名简洁约束固化到现行结构治理中，后续新增目录继续受约束。

## 3. 已达成结果

通过 `P11.10.5`，当前已达成：

1. UI-facing model 不再直接依赖正式 contract DTO。
2. `src/adapters/` 已成为真实适配层。
3. 正式链路已收口为：
   - `contract DTO -> adapter normalized model -> runtime/service model -> UI view model -> component props`
4. 页面和展示组件不再承担 DTO 归一化职责。
5. 目录命名简洁约束继续有效，当前批未新增违反约束的长期目录。

## 4. 未在本计划内继续扩展的内容

以下内容不再算作 `P11.8` 本体：

1. 真实 UI refresh
2. 新版后端接口契约接轨
3. 大规模目录重命名或清仓
4. 新的共享壳视觉改造

若后续需要继续扩展这些内容，统一在 `P11.10` 后续批次或新计划内处理。
