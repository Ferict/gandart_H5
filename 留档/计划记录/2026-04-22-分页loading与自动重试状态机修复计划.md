## 1. 目标

1. 让分页 loading 动画只在真实分页加载链路运行时出现。
2. 首页藏品、公告、个人藏品三处 footer 不再把“非错误、非到底”的稳定状态默认渲染成 loading。
3. 分页失败进入自动重试，重试耗尽后转错误。
4. 分页成功但无新增进入无进展重试，重试耗尽后按现行口径转 endline 或 error。
5. 修复首页藏品追加进入动画无挂载队列时可能不释放 `entering` 的风险。

## 2. 范围

1. 新增首页 page-local shared 分页 loading 状态机。
2. 改首页藏品远端分页返回值，从布尔值改为结构化结果。
3. 改个人藏品远端分页返回值，从布尔值改为结构化结果。
4. 公告非首屏刷新接入同一自动重试口径。
5. 三处 footer mode 改为由真实分页链路、错误或 endline 决定。
6. 补状态机、远端分页、展示层和 activity 远端状态测试。

## 3. 不做什么

1. 不改 `HomeRailListLoadingFooter` 的视觉结构和 mode 枚举。
2. 不调整个人中心视觉、二维码、密钥、资产总额等样式问题。
3. 不改正式后端接口契约。
4. 不读取或移动 `垃圾桶/`。
5. 不执行任何删除操作。
6. 不进入首页整体简化或真实后端假对接重构。

## 4. 停止边界

1. 发现需要改变正式 API contract 时停止。
2. 发现需要新增长期目录或新留档分类时停止。
3. 发现需要真实后端、secret、账号或设备才能继续时停止。
4. 发现质量检查失败且无法判断是否由本批引入时停止。
5. 完成本批代码、测试、留档回写、验证和提交边界后停止。

## 5. 失败退出条件

1. `npm run type-check`、`npm run test:unit -- tests/unit/home`、`npm run lint:eslint`、`npm run lint:style`、`npm run lint:format`、`npm run lint:docs` 任一失败且无法在本批范围内修复。
2. `npm run build:h5` 因本批代码失败且无法修复。
3. `npm run build:app` 因本地平台链缺失失败时，不扩大修复范围，只在交付记录写明环境限制。
4. 工作树出现影响本批判断的非本轮冲突改动。

## 6. 验收标准

1. `.home-list-loading-footer.is-loading` 可见时，能追踪到当前有效分页链路。
2. 三处 footer 在稳定、有更多但未触发分页时不显示 loading。
3. 错误自动重试 3 次；无进展自动重试 2 次。
4. 首页藏品、公告、个人藏品相关单测通过。
5. 计划内质量命令完成并记录结果。
6. 审查记录和交付记录完成回链。
