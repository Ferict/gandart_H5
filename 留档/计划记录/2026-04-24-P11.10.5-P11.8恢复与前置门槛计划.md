## 1. 本批目标

本批只做四件事：

1. 填实 `src/adapters/`，让 `home / activity / profile / detail` 的 DTO 归一化正式落到 adapter 层。
2. 切断 `models/*` 对 `content-api.contract.ts` 的直接依赖，改为内部稳定模型。
3. 让 `services/*` 只做编排和调用，不再自己内嵌大段 DTO 映射。
4. 补最小测试保护，确保这一层边界后续不会回退。

## 2. 本批不做的事

1. 不改 `.vue` 模板和样式输出。
2. 不改正式后端 contract 结构。
3. 不改 provider seam。
4. 不做真实 UI refresh。
5. 不做目录大搬迁或清仓。

## 3. 子代理执行口径

本批按补充计划执行：

1. 已优先拆出互不重叠的 adapter 写入面，由子代理承担。
2. 共享壳、共享组件、主入口、留档与主线状态回填继续由主控独占。
3. 若后续因补测或重开子批继续扩展本批，仍默认先按 ownership 拆子代理，而不是主控整批串行。

## 4. 新版后端契约兼容口径

本批新增的 adapter、normalized model 与 runtime/service 边界，都只服务于前端稳定模型，不把当前 formal contract 重新固化为 UI 真值。  
后续若切换新版后端接口契约，优先替换 adapter 输入侧与 mapper，不再次打穿 `models / pages / templates`。
