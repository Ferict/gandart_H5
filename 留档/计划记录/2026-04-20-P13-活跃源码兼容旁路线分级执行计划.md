## 1. 目标

1. 固化 P13 对活跃源码兼容旁路线的分级结果。
2. 明确哪些对象必须保留，哪些对象冻结不扩展，哪些对象待拍板。
3. 阻止把仍注册、仍被导航链引用的次级页入口误判为可清理死链。
4. 给后续执行批次定义停止边界、失败退出条件和汇报要求。

P13.2 修正：

1. “仍注册、仍被导航链引用”只代表直接移走有风险。
2. 它不能证明对象有业务价值。
3. 已挂线但只有 construction redirect 的对象，应进入 P13.2 挂线垃圾治理计划。

## 3. 当前分级

### 3.1 `keep`

1. `src/pages/home/`
2. `src/pages/profile-asset-detail/`
3. `references/README.md`

处理原则：

1. 本轮不动。
2. 后续如涉及页面结构或内容消费，只能进入独立实现批次。

### 3.2 `freeze`

1. `src/pages/updating/`
2. `src/pages/notice-detail/`
3. `src/pages/action-entry/`
4. `src/pages/content-resource/`
5. `src/pages/profile-address/`
6. `src/pages/profile-assets/`
7. `src/pages/service-entry/`
8. `src/pages/settings/`
9. `src/services/page-entry/`
10. `src/ports/pageContent.port.ts`
11. `src/adapters/pageContent.mock.adapter.ts`

处理原则：

1. 保留现行兼容职责。
2. 不继续扩展新的业务职责。
3. 不自动转垃圾桶。
4. 后续只有在替代链、路由注册、README 和口径全部更新后，才能重新审查是否转移到 `垃圾桶/`。

### 3.3 `defer`

1. `resolveNoticeContent`
2. notice preset 相关 mock 数据和 `PageContentPort` notice 方法

处理原则：

1. 暂不动源码。
2. 等用户确认 `notice-detail` 后续形态后再决定：
   - 保留为冻结预留。
   - 接入占位公告详情页。
   - 写明确计划后转移到 `垃圾桶/`。

### 3.4 `trash`

当前无对象满足 `trash` 条件。

## 4. 执行顺序

推荐按两步处理：

1. 第一步：完成本计划与审查记录落盘，回写主线规划，形成 docs-only 提交。
2. 第二步：如用户确认继续执行 P13 后续批次，只处理 `defer` 项的策略选择，不动 `freeze` 项。

如进入第二步，必须先确认：

1. `notice-detail` 是否继续只是 construction redirect。
2. 是否需要保留 notice placeholder preset 能力。
3. 若不保留，目标 `垃圾桶/` 落点是什么。

## 5. 验收标准

本批 docs-only 验收：

1. P13 审查记录已落盘。
2. P13 执行计划已落盘。
3. 主线规划已回写 P13 状态和索引。
4. 未执行删除。
5. 未转移文件到 `垃圾桶/`。
6. `git diff --check` 通过。
7. `npm run lint:format` 通过。
8. `npm run lint:docs` 通过。
9. Git 提交已形成，或交付记录写明暂缓原因。

## 6. 待拍板项

1. `notice-detail` 后续是继续 redirect 到 `updating`，还是需要恢复独立公告详情占位页。
2. `resolveNoticeContent` 和 notice preset mock 数据是否继续保留为冻结预留。
3. 若要转移 `defer` 项到 `垃圾桶/`，是否允许建立对应冻结落点。

## 7. 停止边界

任一情况出现必须停止：

1. 需要改动 `pages.json`。
2. 需要改动任何页面行为。
3. 需要执行删除或等价删除命令。
4. 需要把文件移动到 `垃圾桶/`，但缺少用户确认或明确计划。
5. 发现某个冻结入口仍有未登记真实业务职责。
6. 需要新增长期目录。
7. 工作树出现非本轮产生的 dirty 文件。

## 8. 失败退出条件

1. 文档检查失败且无法在 docs-only 范围内修复。
2. P13 计划和实际源码结构不一致。
3. `pages.json` 注册入口与页面文件存在性不一致。
4. `homeRailNavigation.service.ts` 仍构建的入口被误判为可转垃圾桶。
5. 用户要求直接删除文件或目录，但未完成二次确认和垃圾桶替代方案。

## 9. 完成后汇报要求

本批完成后必须汇报：

1. P13 完成到哪一步。
2. `keep / freeze / defer / trash` 结论。
3. 是否有源码改动。
4. 是否有删除或垃圾桶转移。
5. 验证命令和结果。
6. 交付记录路径。
7. Git 提交索引。
8. 下一步建议和待拍板项。

## 10. 当前不做什么

1. 不把冻结入口当废弃死链。
2. 不做页面真实化。
3. 不做 UI 或窗口宽度实修。
4. 不把 `page-entry` 打散到各页面目录。
5. 不把 `pageContent` 升格为正式后端 API。
6. 不执行任何删除或转移。
