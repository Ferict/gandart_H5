## 6. 二线需补信息

主线开工前需要二线补充：

1. 当前是否有未提交或未回传的页面级 UI 改动。
2. 当前实际写入文件。
3. 当前只读未改文件。
4. 当前 Git 分类。
5. 当前字段缺口。
6. 当前未确认后端契约项。
7. 当前是否触碰或需要触碰主线禁区文件。
8. 当前是否需要主线新增 adapter、runtime、service 或 view-model 字段。
9. 当前是否需要视觉证据包。

如果没有，二线按固定模板回传“无二线页面改动、无字段缺口、等待授权”。

## 7. 停止条件

3. 发现需要改正式后端契约正文。
4. 发现需要改视觉效果或交互语义。
5. 文档检查失败且原因超出 docs-only 范围。
6. Git 边界无法保持 docs-only。
