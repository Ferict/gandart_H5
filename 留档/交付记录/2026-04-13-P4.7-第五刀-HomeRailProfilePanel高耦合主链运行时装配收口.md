## 2. 收口结果

1. `HomeRailProfilePanel` 不再直接持有下列高耦合装配链：
   - `useProfileScenePatchController(...)`
   - `useHomeRailProfileAssetDataPipeline(...)`
   - `useHomeRailProfileViewState(...)`
   - `useProfileAssetVisualReveal(...)`
   - `useProfileAssetResultWindow(...)`
   - `useProfileAssetSearchRevealTransition(...)`
   - `useProfileAssetReloadCoordinator(...)`
   - `useProfilePagePresentationRuntime(...)`
   - `useHomeRailProfileLoadMore(...)`
   - `useHomeRailProfileTopPresentation(...)`
   - `useHomeRailProfileContentLifecycle(...)`
   - `useHomeRailProfileEffects(...)`
2. 新 `useHomeRailProfilePanelRuntime` 统一承接：
   - profile scene patch
   - asset query 与 remote list pipeline
   - 结果区 window / view state / visual reveal
   - search reveal 与 reload coordinator
   - page presentation runtime
   - load-more / top presentation
   - refresh / activation / visible-update lifecycle
   - watcher/effects 装配
3. 父页继续保留：
   - 顶部动作与藏品详情导航
   - `profileAssetResultsStageRef` 的 template-ref adapter
   - 模板结构与 `defineExpose`

## 3. 风险控制

1. 只搬接线，不改既有 profile rail 运行时模块接口。
2. 导航、header 交互和模板 adapter 继续留在父页，避免新 orchestrator 过度扩张。
3. `activeCategory` 与 `profileSummaryAddress` 仍对父页显式暴露，方便后续读代码时定位导航依赖。
