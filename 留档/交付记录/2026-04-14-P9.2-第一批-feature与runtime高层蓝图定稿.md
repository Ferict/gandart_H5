## 1. 定稿结论

### 1.1 继续保留的稳定锚点

以下锚点继续保留，不在本阶段重命名或拆顶层：

1. `src/pages/`
2. `src/services/`
3. `src/models/`
4. `src/components/`
5. `src/composables/`
6. `src/contracts/`
7. `src/ports/`
8. `src/implementations/`
9. `src/adapters/`
10. `src/mocks/`
11. `src/stores/`
12. `src/utils/`
13. `src/static/`

### 1.2 高层语义分层

后续整编统一按以下视角解释现有锚点：

1. 页面壳与 page-local runtime
   - `src/pages/*`
2. 共享 UI 与共享前端运行时
   - `src/components/`
   - `src/composables/`
   - `src/utils/`
   - `src/stores/`
3. 领域级共享运行时
   - `src/services/*`
   - `src/models/*`
4. 内容接入链
   - `src/contracts/`
   - `src/ports/`
   - `src/implementations/`
   - `src/adapters/`
5. mock 与开发数据支撑
   - `src/mocks/`

### 1.3 后续优先迁移候选

如果后续继续做更高层整编，优先候选固定为：

1. `pages/home` 与 `services/home-rail` 的高层边界再澄清
2. `pages/profile-asset-detail` 与 `services/profile-asset-detail` 的高层边界再澄清
3. `services/page-entry` 的继续收缩或归位

### 1.4 明确不扩展项

本阶段明确不做：

1. 新建 `src/features/`
2. 让 `services/*` 承接页面私有装配
3. 让 `pages/*` 继续回流真正跨页共享组件
4. 对内容接入链做大范围拆并

## 2. 后续执行口径

下一步若进入 `P9.3` 试点，只允许选择：

1. 有明确高层归属收益
2. 迁移范围可控
3. 不会引入新的顶层目录
4. 可由 `type-check` 和现有单测直接兜底

不满足以上四条的候选，统一延后。
