## 2. 收口结果

1. `useHomeRailActivityPanelRuntime` 不再直接持有：
   - `useActivityPagePresentationRuntime(...)`
   - `useHomeRailActivityNoticeReload(...)`
   - `useHomeRailActivityContentLifecycle(...)`
2. 新 `useHomeRailActivityRefreshRuntime` 统一承接：
   - notice list reload / retry
   - refresh presentation runtime
   - staged notice list update 管理
   - refresh / activation / visible-update lifecycle
3. `useHomeRailActivityPanelRuntime` 继续保留：
   - scene patch
   - data pipeline
   - result window
   - visual reveal
   - navigation
   - presentation state
   - effects

## 3. 风险控制

1. 本刀只搬装配层，不改 notice reload、presentation runtime、content lifecycle 三个组合器的实现。
2. helper 输入类型直接复用既有组合器参数签名，不在 activity 主链里新增第三套类型口径。
3. `HomeRailActivityPanel` 的对外暴露接口未改变，仍保持 `refreshContent()` 与 `waitForRefreshPresentation()`。
