## 1. 背景

用户通过旧前端和新前端截图确认：旧前端“我的/个人中心”顶部的“区块链地址”不是新前端当前临时展示的用户 id 或短地址文案，真实情况下应对应旧前端用户信息中的区块链地址字段。

本记录作为正式留档。主线后续按 [10-用户最终要求口径.md](/H:/工作/天异/uniapp+vue新架构/规则体系/口径/10-用户最终要求口径.md#9-个人中心区块链地址最终口径) 第 9 节接入。

## 2. 用户最终裁定

1. 个人中心顶部区块链地址真实接旧前端时，优先使用旧用户信息返回 `userInfo.ntf_url`。
2. `userId`、`accountId`、手机号、邀请标识、mock address 和 UI 短地址文案不得作为个人中心区块链地址真实来源。
3. `user_hash` / `user_hash_text` 属于藏品详情、寄售详情等藏品或合约地址展示链，不等同于个人中心顶部用户区块链地址。
4. 当 `ntf_url` 缺失时，前端可以展示“拥有藏品后自动生成”类占位文案，但不得把占位文案写回、复制或作为身份参数。

## 4. 主线接入要求

1. 主线接入个人中心地址时，先读取上述现行口径，再改前端实现。
2. 接入层可以在 adapter 内兼容 `wallet_address`、`chain_address`、`blockchain_address` 等未来标准字段，但优先级不得高于已确认的旧前端 `ntf_url`。
3. `content` 用户态请求不得新增 `userId`、`accountId` 或 `walletAddress` 作为身份参数；用户身份继续来自登录态、token 或 session。
4. 新前端当前临时展示的短地址只允许作为过渡 UI，不得被主线当作最终真实字段来源。
