## 2. 收口结果

1. `HomeRailHomePanel` 不再直接持有下列高耦合装配顺序：
   - `useHomeRailHomeMarketDataPipeline(...)`
   - `useHomeMarketResultWindow(...)`
   - `useHomeHomeVisualReveal(...)`
   - `useHomePagePresentationRuntime(...)`
   - `useHomeScenePatchController(...)`
   - `useHomeRailHomeMarketReload(...)`
   - `useHomeRailHomeContentLifecycle(...)`
   - `useHomeRailHomePresentationState(...)`
   - `useHomeRailHomeEffects(...)`
2. 新 `useHomeRailHomePanelRuntime` 统一承接：
   - market query 与 remote list pipeline
   - market result window
   - visual reveal
   - page presentation runtime
   - scene patch
   - market reload
   - refresh / activation / visible-update lifecycle
   - home rail presentation state
   - watcher/effects 装配
3. 父页继续保留：
   - 内容状态与页面运行时状态初始化
   - 点击导航链路
   - template-ref adapter
   - 模板结构与 `defineExpose`

## 3. 风险控制

1. 这刀只搬接线，不改已有运行时模块对外接口。
2. 结果区 section、导航链路和模板 adapter 继续留在父页，不把所有内容压进一个新黑盒。
3. `market` 主链内部仍然复用原有 page-local composable，不新造 shared 抽象。
