## 1. 问题

用户复核指出：当前运行时几乎看不到分页 loading 动画。

复查结论：分页链路已经存在，但本地 `mock` provider 返回过快，`requesting -> committing -> idle` 可能在浏览器绘制前结束，导致 `.home-list-loading-footer.is-loading` 没有可见帧。

## 2. 修复

1. `useHomeRailPaginationLoadingChain` 在分页触发后立即启动真实 attempt。
2. 状态机保持 `requesting` 至少经过一次 DOM/帧刷新，让 footer 有机会绘制 loading。
3. `requestAnimationFrame` 不可用时使用 `16ms` 帧兜底。
4. 补充单元测试，确认快速 resolve 的 attempt 也会保持一帧 `requesting` 与 `isLoading=true`。
5. 补齐 activity fake-timer 测试的 `requestAnimationFrame` stub，避免测试环境卡在帧等待。

## 3. 边界

1. 不恢复“有更多就默认显示 loading”的旧逻辑。
2. loading 仍只由当前有效分页链路驱动。
3. 本次不处理真实 HTTP 假后端服务；该问题仍由后端 API 假对接缺口记录承接。
