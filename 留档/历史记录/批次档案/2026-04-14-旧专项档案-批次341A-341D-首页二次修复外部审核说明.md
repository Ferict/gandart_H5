### 批次341B：抽屉延迟跳转残留风险修复

问题：

- 抽屉跳转仍是裸 `setTimeout`
- 组件在延迟期间销毁时，可能继续触发残留导航

修复：

- `HomeShellDrawer.vue` 新增 `activateTimerId`
- 新增 `clearActivateTimer()`
- 组件卸载时执行 `clearTimeout`
- 跳转前增加最小存活性保护

代码前后对照：

修复前：

```ts
const handleEntryActivate = (routeUrl: string) => {
  emitClose()
  setTimeout(() => {
    uni.navigateTo({ url: routeUrl })
  }, 40)
}
```

修复后：

```ts
let activateTimerId: ReturnType<typeof setTimeout> | null = null
let isComponentAlive = true

const clearActivateTimer = () => {
  if (!activateTimerId) {
    return
  }
  clearTimeout(activateTimerId)
  activateTimerId = null
}

const handleEntryActivate = (routeUrl: string) => {
  clearActivateTimer()
  emitClose()
  activateTimerId = setTimeout(() => {
    if (!isComponentAlive) {
      return
    }
    activateTimerId = null
    uni.navigateTo({ url: routeUrl })
  }, 40)
}
```

结果：

- 抽屉延迟跳转已具备句柄管理
- 组件销毁后不会再继续触发残留导航

对应提交：

- 子仓：`2671a13`
- 根仓：`318c590`

### 批次341C：banner focalPoint 契约与运行时边界收口

问题：

- `focalPoint.x/y` 只写成 `number`
- 页面层直接做 `x * 100 / y * 100`
- 后端若下发 `50` 而不是 `0.5`，会直接生成 `5000%`

修复：

- `content-api.contract.ts` 明确 `focalPoint` 为 `0~1` 归一化坐标
- `统一内容接口契约.md` 补齐正式口径
- `HomeRailHomePanel.vue` 新增 `clampNormalizedFocalPointUnit()`
- 非法值统一回退到安全值

代码前后对照：

修复前：

```ts
const resolveBannerImageStyle = (item: HomeBannerItem): CSSProperties => {
  if (!item.focalPoint) {
    return { objectPosition: '50% 50%' }
  }

  return {
    objectPosition: `${item.focalPoint.x * 100}% ${item.focalPoint.y * 100}%`,
  }
}
```

修复后：

```ts
const clampNormalizedFocalPointUnit = (value: number) => {
  if (!Number.isFinite(value)) {
    return 0.5
  }
  if (value < 0) {
    return 0
  }
  if (value > 1) {
    return 1
  }
  return value
}

const resolveBannerImageStyle = (item: HomeBannerItem): CSSProperties => {
  if (!supportsBannerFocalPointObjectPosition || !item.focalPoint) {
    return {}
  }

  const normalizedX = clampNormalizedFocalPointUnit(item.focalPoint.x)
  const normalizedY = clampNormalizedFocalPointUnit(item.focalPoint.y)

  return {
    objectPosition: `${(normalizedX * 100).toFixed(2)}% ${(normalizedY * 100).toFixed(2)}%`,
  }
}
```

结果：

- `focalPoint` 输入已经有明确契约
- 页面层不再直接信任后端数值
- 不会再输出异常 `object-position`

对应提交：

- 子仓：`91fd474`
- 根仓：`2a3e67c`

### 批次341D：banner 焦点裁切 H5 / 非 H5 保证范围说明

- 非 H5 端未给出正式保证范围和降级口径

修复：

- 页面层仅在 H5 端启用 `object-position`
- 非 H5 端统一降级为中心裁切安全路径
- 本文档与 `统一内容接口契约.md` 明确写明当前正式保证范围

结果：
