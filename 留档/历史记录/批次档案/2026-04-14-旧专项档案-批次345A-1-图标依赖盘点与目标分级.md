## 三、目标分级

### P0：壳层核心图标，优先迁移

- `homeShellIconMap.ts`
- `HomeRailTopbar.vue`
- `HomeShellDrawer.vue`
- `HomeShellNavRail.vue`
- `HomeShellTabbar.vue`

原因：

- 这些图标命中首页主壳与全局入口
- 在 `APP 打包壳 / 手机 Web / 电脑 Web` 三场景中曝光率最高
- 一旦 App 端出现丢图、尺寸异常或渲染性能问题，这批位置最先暴露

### P1：首页三主面板高频业务图标，第二批迁移

- `HomeRailHomePanel.vue`
- `HomeRailActivityPanel.vue`
- `HomeRailProfilePanel.vue`

原因：

- 暴露频率高
- 首页是当前逐页校对主阵地
- 但图标数量和场景复杂度高于壳层，应晚于壳层迁移

### P2：正式页头部与高频动作图标，第三批迁移

- `settings/index.vue`
- `service-entry/index.vue`
- `action-entry/index.vue`
- `content-resource/index.vue`
- `notice-detail/index.vue`
- `profile-address/index.vue`
- `profile-assets/index.vue`
- `profile-asset-detail/index.vue`

原因：

- 这些页面已有正式承接链路
- 图标以返回、菜单、搜索、复制、二维码等高频动作为主
- 可在壳层与首页主面板迁完后批量接入统一适配层

### P3：占位/装饰图标，最后处理

- `updating/index.vue`
- `content-resource/index.vue` 中占位类图标
- 首页/活动/个人中心中的占位与装饰图标

原因：

- 对主流程可用性影响最低
- 可等核心导航与高频业务图标稳定后再统一收口

## 四、后续迁移约束

- 不允许页面层继续直接 `import { ... } from 'lucide-vue-next'`
- 必须先落统一适配层 `AetherIcon.vue`
- 已拍板后端方案固定为 `Iconfont`
- 适配层必须使用编译时宏隔离：
  - `H5 / 电脑 Web` 保留 `lucide`
  - `APP 打包壳` 走 `Iconfont`
- 不允许在适配层里保留运行时三后端 `v-if / v-else` 混合分发

## 五、执行顺序建议

1. `345A-2`：先建 `AetherIcon.vue`
