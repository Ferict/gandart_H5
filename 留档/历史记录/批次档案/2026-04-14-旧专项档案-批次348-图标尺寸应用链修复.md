# 批次348-图标尺寸应用链修复

## 目标

修复 `AetherIcon` 的 H5 尺寸参数未正确生效问题。

## 问题

- `AetherIcon.vue` 已向 H5 lucide 组件透传 `size`
- 但同文件样式又对 `.aether-icon` 与 `.aether-icon--svg` 强制写 `width/height: 1em`
- 导致 H5 图标最终回退为父级字号驱动，而不是 `size` 驱动

## 修复

- 从共享根类 `.aether-icon` 移除 `width/height: 1em`
- H5 分支仅保留 `line-height: 0` 与 `svg` 基础绘制样式
- `1em` 尺寸约束只保留在 `.aether-icon--font`，不再污染 H5 SVG 分支

## 结果

- H5 分支恢复由 `size` 参数直接控制图标尺寸
- App / Harmony 字体图标分支维持现有 `fontSize` 驱动，不受本批影响

## 验证

- `npm run type-check` 通过
- `npm run build:h5` 通过
- `npm run build:app` 通过
- `npm run build:app-harmony` 通过
