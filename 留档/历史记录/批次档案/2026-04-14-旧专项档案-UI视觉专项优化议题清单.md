## 1. 文档定位

- 本文不是执行记录。
- 本文用于把后续 UI 视觉专项优化的议题先收成清单，避免后面边做边漏。

## 2. 优先议题

1. 共享顶部与底部 chrome 的视觉基线
2. 毛玻璃、透明层、fallback 与性能边界
3. 热区、视觉壳、高亮壳的一致性
4. 页面标题体系与英文辅助标题体系
5. 首页、活动页、个人中心、设置页、公告详情页、更新中页的统一风格收敛
6. 平板竖屏与宽屏舞台下的视觉密度一致性
7. 鼠标 / 触摸双路径下的交互反馈一致性

## 3. 执行口径

1. 后续按页面逐页审查
2. 每页都走：
   - 用途确认
   - 需求确认
   - 审查
   - 计划
   - 落地
   - 复查
   - 补注释
   - 对接口
3. 不允许跨多个页面随手混改

## 4. 待决事项

1. 是否首批就引入统一动画 token
2. 是否首批建立共享视觉基础组件（如统一 badge、tab、toolbar）
3. 是否先做首页与个人中心，再做其余页面
