# 审核：视口与键盘细则正文落地

确认是否已经具备把“视口与键盘”从研究结论升级成正式细则正文的条件，并划清本批次允许修改的范围。

## 3. 当前判断

1. 研究资料已经足够，视口与键盘不再停留在“问题矩阵”阶段。
2. 当前最需要正式成文的是：
   - `viewport-fit=cover`
   - safe-area 消费边界
   - `vh / svh / lvh / dvh`
   - `VisualViewport`
   - 键盘与输入区职责边界
3. 这批如果不先写细则，后续无论做共享壳层还是页面模板，都容易把：
   - 宿主问题
   - H5 视口问题
   - 页面滚动问题
     混成同一类补丁。

## 4. 兼容性判断

1. 本批次只动文档层，不会影响旧仓冻结状态。
2. 本批次不改新仓源码，不会对后续源码骨架造成回滚压力。
3. 本批次与现行主规范兼容，属于细则补强，不会推翻已拍板的环境与工具链基线。
