# 审核 - 多端适配问题矩阵与解决路径研究

## 1. 审核背景

- 用户要求先系统梳理 iOS / Android App、iOS / Android / 桌面 Web 在 UI 适配层的常见问题。
- 要求先列问题，再查官方解决路径，并判断这些方案是否适配 `uni-app + Vue`。

## 2. 审核范围

- 顶部安全区、状态栏、导航栏
- 底部安全区、标签栏、home indicator
- 视口、键盘、滚动、fixed / sticky
- 鼠标 / 触摸 / 轨迹板交互差异
- 热区、焦点、高亮、动画
- WebView / 宿主壳 / 前端责任边界

## 3. 审核结论

1. 这批问题不能只靠前端页面规范解决，必须同时区分：
   - 前端可控问题
   - 客户端壳工程必须配合的问题
   - 可以用后端配置辅助，但不能靠后端“修 UI”的问题
2. 当前最重要的不是马上写更多页面规则，而是先建立统一问题矩阵和责任边界。
3. 经官方资料核对，后续细则至少要覆盖：
   - Safe area 与 edge-to-edge
   - viewport / keyboard / VisualViewport
   - pointer / hover / touch-action
   - text inflation
   - backdrop-filter fallback
   - scroll container / sticky / scroll-view
   - Web 宽屏主列与平板竖屏 token 放大

## 4. 本批不做

- 不修改源码
- 不立即补所有细则正文
- 不实现宿主壳能力

2. 再按文档拆细则专题
