# 全局阴影统一规范与迁移计划

## 目标

- 本规范用于统一全仓 `box-shadow`、`text-shadow`、玻璃投影和强调发光。
- 统一口径固定为：轻、淡、柔、不偏移。
- 所有正式阴影 token 统一使用 `x = 0`、`y = 0`，不再通过向下偏移制造“漂浮感”。

## 阴影语义表

| 语义         | 变量                            | 数值                                | 适用场景                                      |
| ------------ | ------------------------------- | ----------------------------------- | --------------------------------------------- |
| 普通表面-轻  | `--aether-shadow-surface-sm`    | `0 0 16px rgba(15, 23, 42, 0.04)`   | 小型白卡、预览壳、次级白色操作壳              |
| 普通表面-中  | `--aether-shadow-surface-md`    | `0 0 24px rgba(15, 23, 42, 0.05)`   | 主卡、信息卡、详情白卡、主要白色容器          |
| 浮层-面板    | `--aether-shadow-overlay-sheet` | `0 0 32px rgba(15, 23, 42, 0.08)`   | 抽屉、筛选面板、侧滑/底部面板                 |
| 浮层-悬浮    | `--aether-shadow-overlay-float` | `0 0 24px rgba(15, 23, 42, 0.08)`   | 悬浮胶囊、玻璃 tabbar、黑色悬浮按钮、浮动提示 |
| 强调发光     | `--aether-shadow-accent-cyan`   | `0 0 16px rgba(34, 211, 238, 0.18)` | 青色扫描线、品牌强调发光                      |
| 顶栏滚动态   | `--aether-shadow-topbar-scroll` | `0 0 16px rgba(15, 23, 42, 0.04)`   | 顶栏滚动吸附态                                |
| 图上白字阴影 | `--aether-text-shadow-on-image` | `0 0 2px rgba(15, 23, 42, 0.18)`    | 仅限图上白字或深浅背景不可控的白字角标        |

## 兼容映射

- `--aether-shadow-soft -> --aether-shadow-surface-md`
- `--aether-shadow-soft-xs -> --aether-shadow-surface-sm`
- `--aether-glass-shadow -> --aether-shadow-overlay-float`

说明：

- 旧变量名继续保留，作为兼容别名存在。
- 所有 `var(--aether-shadow-*, fallback)` 的 fallback 必须与正式 token 完全一致。
- 不允许出现“token 定义一套、fallback 另一套”的漂移状态。

## 允许场景

- 普通白色卡面、信息卡、预览壳、小型白色容器，只允许使用 `surface-sm` 或 `surface-md`。
- 抽屉、日期筛选面板、侧滑层、底部浮层，只允许使用 `overlay-sheet`。
- 悬浮胶囊、悬浮黑色按钮、玻璃 tabbar、玻璃指示器，只允许使用 `overlay-float`。
- 青色扫描线或品牌发光，只允许使用 `accent-cyan`。
- 顶栏滚动态，只允许使用 `topbar-scroll`。
- `text-shadow` 默认禁用，仅允许图上白字场景使用 `--aether-text-shadow-on-image`。
- `hover` 态如保留阴影变化，只允许从当前语义上浮一档，并且只在 `(hover: hover) and (pointer: fine)` 生效。

## 禁止场景

- 禁止普通页面层继续直写 `0 0 24px rgba(...)`、`0 0 28px rgba(...)`、`0 20px 50px rgba(...)` 等字面量阴影。
- 禁止通过 `y` 轴正偏移制造“下坠投影”。
- 禁止 `active` 态新增阴影或把阴影做重。
- 禁止 `focus-visible` 依赖阴影表达焦点，焦点仅允许使用描边或 ring。
- 禁止把强调发光当普通表面阴影使用。
- 禁止 `text-shadow` 扩散到常规白底正文、列表、卡面文字。

## 旧值到新语义映射表

| 旧写法                               | 新语义                 | 说明                       |
| ------------------------------------ | ---------------------- | -------------------------- |
| `0 0 24px rgba(15, 23, 42, 0.05)`    | `surface-md`           | 普通主卡、白卡统一值       |
| `0 0 18px rgba(15, 23, 42, 0.04)`    | `surface-sm`           | 统一收口到 `16px / 0.04`   |
| `0 0 28px rgba(15, 23, 42, 0.05)`    | `surface-md`           | 过强白卡阴影回落           |
| `0 0 28px rgba(15, 23, 42, 0.06)`    | `surface-md`           | 白卡不再单独抬高阴影       |
| `0 20px 50px rgba(0, 0, 0, 0.08)`    | `overlay-sheet`        | 底部/筛选面板取消偏移投影  |
| `0 14px 30px rgba(15, 23, 42, 0.18)` | `overlay-float`        | 悬浮胶囊取消重投影         |
| `0 0 20px rgba(15, 23, 42, 0.12)`    | `overlay-float`        | 黑色悬浮按钮回到柔和浮层值 |
| `0 0 14px rgba(34, 211, 238, 0.24)`  | `accent-cyan`          | 品牌发光收口到统一青色发光 |
| `0 1px 2px rgba(15, 23, 42, 0.18)`   | `text-shadow-on-image` | 改为无偏移图上白字阴影     |

## 特例清单

- `box-shadow: none` 允许保留。
- `inset` 阴影允许作为玻璃高光或边框模拟保留，但不能替代正式阴影语义。
- `0 0 0 2px ...` 这类 focus ring 或指示点描边不算普通阴影，不纳入本规范的表面/浮层分档。
- 组件内部纯动画过渡里的 `box-shadow 180ms ease` 可以保留，不属于数值语义层。

## 两阶段迁移规则

### 第一阶段：建立语义和兼容层

- 在 `src/uni.scss` 中建立正式 token。
- 旧变量改为正式 token 别名。
- 全仓先修正 fallback 漂移，保证 token 与 fallback 同值。

### 第二阶段：按语义分批替换

- 普通白卡统一切到 `surface-sm` / `surface-md`。
- 抽屉、筛选面板、底部悬浮胶囊统一切到 `overlay-sheet` / `overlay-float`。
- 青色发光切到 `accent-cyan`。
- 顶栏滚动态切到 `topbar-scroll`。
- `text-shadow` 仅保留批准场景，其余移除。
- `active` 态移除阴影加重写法。

## 验收规则

- 不再存在 token 值与 fallback 值不一致。
- 普通表面卡面不再直写近似阴影字面量。
- 顶栏滚动态不再使用向下偏移阴影。
- `text-shadow` 只剩批准场景。
- `active` 态不再新增或加强阴影。
- 抽样页面能明确归类到普通表面、浮层、强调、顶栏、文本特例五类语义。
