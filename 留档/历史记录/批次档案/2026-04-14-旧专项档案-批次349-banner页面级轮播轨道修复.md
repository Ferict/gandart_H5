# 批次349-banner页面级轮播轨道修复

## 目标

将首页 banner 从“内容容器内裁切的轮播”改为“页面级轨道轮播”，让下一张卡片从页面外进入，而不是在页面内边距容器里被提前裁切。

## 根因

- `home-panel` 自身带左右内边距
- `home-banner-carousel` 原先仍只占内边距内容盒宽度
- 结果是 banner 轨道被裁在内容盒内，下一张卡片只能在内边距容器内出现

## 修复

- 扩大 `home-banner-carousel` 到页面宽度：
  - `width: calc(100% + (padding * 2))`
  - `margin-left: calc(-1 * padding)`
- 比例壳改为：
  - 宽度只按可见卡片宽度计算
  - `margin: 0 auto`
  - `overflow: visible`
- 保留当前卡片比例与内容边距，不改数据链

## 结果

- banner 轨道裁切边界从“内边距内容盒”移到“页面宽度”
- 下一张卡片会从页面外进入
- 当前卡片仍按现行可见宽度与比例显示

## 验证

- `npm run type-check` 通过
- `npm run build:h5` 通过
- `npm run build:app` 通过
- `npm run build:app-harmony` 通过
