# 批次385-首页顶部导航投影触发与样式审查

## 审查目标

- 复查首页顶部导航栏滚动后投影链，是未实现、未正确触发，还是无法真正投影到页面上。

## 审查范围

- `src/pages/home/components/HomeRailTopbar.vue`
- `src/pages/home/components/HomeShellTrackStage.vue`

## 审查结果

### 1. 已有滚动状态链，但当前实现不满足“对页面产生投影”

- `HomeShellTrackStage.vue` 已通过 `@scroll` 绑定 `handleTrackScroll($event)`
- `handleTrackScroll()` 里使用 `scrollTop > 0` 将 `homeTrackScrolled` 置为 `true`
- `HomeRailTopbar.vue` 通过 `:is-scrolled="homeTrackScrolled"` 接收状态
- 说明：这条链不是缺失实现

### 2. 当前 `is-scrolled` 样式本身就不是“向下投影”

- 现有样式：`.home-topbar.is-scrolled { box-shadow: 0 0 16px rgba(15, 23, 42, 0.06); }`
- 问题：`x=0, y=0`，这是均匀外发阴影，不是明显向下投影
- 结论：即使滚动态正确命中，当前样式也无法形成“导航栏压到页面内容上”的感受

### 3. 当前滚动态样式只有阴影，没有同步强化顶栏表面层

- `.home-topbar` 基态和滚动态背景几乎一致
- `transition` 虽然声明了 `background-color`，但 `.is-scrolled` 未实际改变背景、边线或表面分离度
- 结果：在 `#fafafa` 背景上，极轻阴影更难读出来

### 4. 触发链在 H5 路径上是接通的，但跨端触发稳定性源码中无法完全自证

- 当前滚动容器是普通 `view` + CSS `overflow-y: auto` + `@scroll`
- 这在 H5 DOM 路径可成立，因为代码同时兜了 `event.target/currentTarget.scrollTop`
- 但对 uni-app 多端来说，它不像 `scroll-view` 那样天然自带统一滚动事件语义
- 结论：
  - “完全没触发”的源码证据不足
  - “跨端触发稳定性不够自证”成立，属于次级风险

## 最终结论

- 这条链不是没做。
- 但从用户要的效果看，当前可以视为“没正确做成”。
- 根因优先级排序：
  1. 样式方向错误：不是向下投影
  2. 投影太轻、表面层未分离
  3. 滚动事件跨端稳定性缺少更强自证
