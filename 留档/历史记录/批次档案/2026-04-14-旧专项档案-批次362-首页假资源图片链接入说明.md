# 批次362-首页假资源图片链接入说明

> 生成时间：2026-03-26
> 范围：首页问题 1，首发藏品卡与市场卡假资源图片链

## 一、问题

首页首发藏品卡与市场卡长期回退到占位图标，不是页面样式问题，而是 mock 内容源仍下发空 `assetId`。

命中点：

- `src/mocks/content-db/drops.ts`
- `src/mocks/content-db/market-items.ts`
- `src/mocks/content-db/assets.ts`

## 二、修复策略

本批不改页面层与 service 层，只闭合假资源链：

1. 在 `assets.ts` 新增首页卡片专用资源记录。
2. 在 `drops.ts` 把首发藏品 `AX-99` 接到 `ASSET-HOME-FEATURED-AX99`。
3. 在 `market-items.ts` 把 `C-01` 到 `C-04` 接到对应 `ASSET-HOME-MARKET-*`。
4. 继续复用现有 `resolveAssetUrl(asset, 'card')`，不新增页面补丁。

## 三、实际接入资源

- `ASSET-HOME-FEATURED-AX99` -> `/static/home/featured/ax99-featured.jpg`
- `ASSET-HOME-MARKET-C01` -> `/static/home/market/c01-market.jpg`
- `ASSET-HOME-MARKET-C02` -> `/static/home/market/c02-market.jpg`
- `ASSET-HOME-MARKET-C03` -> `/static/home/market/c03-market.jpg`
- `ASSET-HOME-MARKET-C04` -> `/static/home/market/c04-market.jpg`

图片尺寸实测均为 `720x720`。

## 四、修改文件

- `src/mocks/content-db/assets.ts`
- `src/mocks/content-db/drops.ts`
- `src/mocks/content-db/market-items.ts`

## 五、验证

已通过：

- `npm run type-check`
- `npm run build:h5`
- `npm run build:app`
- `npm run build:app-harmony`

静态复查：

- 首页首发卡与市场卡已不再命中 `assetId: null`
- 首页卡片资源已具备 `variants.card`

## 六、结论

本批已把首页问题 1 的假资源图片链接通。
页面现在会优先拿到首页卡片图片，不再因为 mock 资源为空而稳定回退到占位图标。
