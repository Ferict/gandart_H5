## 3. 审查结论

1. `result-window` 主链已经过了自然停点，不适合继续沿用 `P4` 的低风险微切策略。
2. 当前最值得继续推进的是首页 `home` 主壳和市场交互链，因为它们仍同时承担：
   - 主模板装配
   - 搜索/排序/标签查询状态
   - 视觉揭示与图片状态
   - 局部结果区装配桥接
3. [useHomeMarketQueryState.ts](/H:/工作/天异/uniapp+vue新架构/src/pages/home/composables/home/useHomeMarketQueryState.ts) 仍然存在明显自然簇，适合作为 `P6` 第一刀候选：
   - sort popover layer / placement / listener
   - active/applied query 同步
   - search reveal 与 debounce 调度
4. [useHomeHomeVisualReveal.ts](/H:/工作/天异/uniapp+vue新架构/src/pages/home/composables/home/useHomeHomeVisualReveal.ts) 的价值在于继续按 `banner / featured / market` 作用域分层，但不应被拆成失去语义的共享 util。
5. [HomeActivityDateFilterSheet.vue](/H:/工作/天异/uniapp+vue新架构/src/pages/home/components/HomeActivityDateFilterSheet.vue) 和 [HomeShellTrackStage.vue](/H:/工作/天异/uniapp+vue新架构/src/pages/home/components/HomeShellTrackStage.vue) 仍然偏厚，但优先级低于 `home` 主壳与市场链。
6. `shared-home-collection-catalog.ts`、`homeRailUpdateCoordinator.service.ts` 等大文件当前更偏 mock/service 主线，不应在本批次重新卷入。
