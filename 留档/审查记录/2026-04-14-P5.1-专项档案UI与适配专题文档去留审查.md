## 3. 审查结论

1. `UI 视觉专项优化议题清单.md` 属于旧专题议题清单，应归为 `A 类历史正文`，迁入 `留档/历史记录/批次档案/`。
2. `多端 UI 适配问题矩阵与解决路径.md` 属于旧专题问题矩阵，应归为 `A 类历史正文`，迁入 `留档/历史记录/批次档案/`。
3. `UI热区页面例外清单.md` 不能直接按普通旧文送历史，因为它仍承担页面级热区例外的当前口径职责。
4. `UI热区页面例外清单.md` 应先吸收到新的现行口径文，再将旧文原位冻结为历史入口。
5. 当前交互与适配稳定规则以 [规则体系/08-交互与适配专题细则.md](/H:/工作/天异/uniapp+vue新架构/规则体系/08-交互与适配专题细则.md) 为准；当前仍有效的例外与执行基线，应新建为 [规则体系/口径/03-交互与适配现行口径.md](/H:/工作/天异/uniapp+vue新架构/规则体系/口径/03-交互与适配现行口径.md)。

## 4. 判定依据

1. `UI 视觉专项优化议题清单.md` 是旧阶段议题收集单，不是现行规则或现行口径。
2. `多端 UI 适配问题矩阵与解决路径.md` 明确自述为“不是最终细则正文”，已有稳定规则 [规则体系/08-交互与适配专题细则.md](/H:/工作/天异/uniapp+vue新架构/规则体系/08-交互与适配专题细则.md) 承接。
3. `UI热区页面例外清单.md` 中登记的例外仍对应当前现行页面，因此必须保留为现行真值源，但不应继续留在旧专项档案路径。
4. 按当前规则，现行口径应进入 `规则体系/口径/`，而不是继续留在 `审核与修改计划/专项档案/`。
