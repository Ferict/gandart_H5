1. `P11.10.1 ~ P11.10.4` docs 治理批已落盘。

## 2. 当前真实耦合点

本轮复查确认，当前最主要的前置门槛缺口不是“页面直接调 HTTP”，而是以下三类结构问题仍然存在：

1. `src/adapters/` 仍为空壳，仅有 `README.md` 与 `.gitkeep`，尚未承接真实 DTO 归一化职责。
2. 多个 runtime-facing model 仍直接 import `src/contracts/content-api.contract.ts` 里的 DTO 或枚举类型，导致 model 层仍与正式契约直接耦合。
3. `services/home-rail/*`、`services/profile-asset-detail/*` 中仍内嵌了大量 DTO -> 内部展示模型的转换逻辑，adapter 层没有成为稳定入口。

## 3. 本批最小安全切口

本批按“零视觉差 + 不扩正式契约 + 不穿透共享壳”口径，最小安全切口固定为：

1. 先建立内部稳定 target/model 类型，切断 `models/*` 对 contract DTO 的直接依赖。
2. 把 home / activity / profile / detail 的 DTO 归一化逻辑下沉到 `src/adapters/content/*`。
3. service 层改为消费 adapter 输出，而不是继续内嵌 DTO 映射。
4. 页面、shared 壳、全局组件模板和样式不作为本批主要修改面；若必须接线，只允许主控做等效接线。

   - ownership：`src/adapters/content/homeRailHomeContent.adapter.ts`、`homeRailActivityContent.adapter.ts`、`homeRailProfileContent.adapter.ts` 及对应 adapter 单测

3. 主控

## 5. 本批停止边界

命中以下任一条件必须停：

1. 需要改正式 contract 结构。
2. 需要改变 provider seam。
3. 需要改页面或组件的可见视觉输出。
4. 需要多个写入代理同时改同一 service / shared 文件。
5. 提交边界开始混乱，无法维持独立代码批。
