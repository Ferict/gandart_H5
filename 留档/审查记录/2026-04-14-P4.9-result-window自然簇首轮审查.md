   - `syncMountedMarketWindow`

### 2.2 `profile` 的自然簇也较清楚

### 2.3 `activity` 当前不是第一优先级

1. `useActivityNoticeResultWindow.ts` 虽然也有相似分层，但总量明显更小。
2. 现在先动 `activity`，收益低于继续处理 `home/profile`。

## 3. 当前判断

1. 下一批高耦合重构的第一刀，不应该从 `activity` 开始。
2. 也不应该一上来直接拆 `switch / replay` 编排，因为那是最脆的行为层。
3. 当前最稳的第一刀应优先切出 `geometry / mounted window / spacer` 这一簇：
   - 读依赖集中
   - 对外协议相对清楚
   - 更容易在不碰行为排序的前提下独立成 page-local helper/runtime

## 4. 建议顺序

1. 先开 `home`：
   - 从 [useHomeMarketResultWindow.ts](/H:/工作/天异/uniapp+vue新架构/src/pages/home/composables/home/useHomeMarketResultWindow.ts) 的 `geometry / mounted window / stage observer` 簇入手
2. 再镜像到 `profile`：

3. `activity` 暂缓，等 `home/profile` 第一刀稳定后再决定要不要镜像

## 5. 结论

1. `P4.9` 若继续实施，第一刀目标固定为：
   - `home` result-window 的 geometry / mounted window / observer 簇
2. 在这一刀完成前，不建议直接重构：
   - `switch / queue / replay`
   - `enter / leave` 时序
3. `profile` 作为第二刀跟进，`activity` 暂不进入本批。
