## 3. 详细事实

### 3.1 当前 contract 文件已经具备的基础

[src/contracts/content-api.contract.ts](/H:/工作/天异/uniapp+vue新架构/src/contracts/content-api.contract.ts) 当前已经具备：

1. 正式资源类型与列表资源类型枚举：
   - `ContentResourceType`
   - `ContentListResourceType`
2. 统一 envelope：
   - `ContentEnvelope<T>`
3. 5 个正式接口的请求 DTO：
   - `ContentSceneRequestDto`
   - `ContentResourceRequestDto`
   - `ContentMarketListRequestDto`
   - `ContentNoticeListRequestDto`
   - `ContentProfileAssetListRequestDto`
   - `NoticeReadActionRequestDto`
   - `ServiceReminderConsumeActionRequestDto`
4. 场景、资源、列表、动作的主要 DTO 结构。

结论：

1. 它已经是当前最接近单一真值源的文件。
2. 继续把真值源放在这里，迁移成本最低。

### 3.2 当前 contract 文件离“后端唯一阅读正文”还差的结构

当前 [src/contracts/content-api.contract.ts](/H:/工作/天异/uniapp+vue新架构/src/contracts/content-api.contract.ts) 仍缺少以下内容：

1. **没有显式 endpoint 总表层**
   - 文件里有 DTO，但没有“5 个正式接口的路径 + 方法 + 对应 request/response 类型”的统一总表。
   - 这些信息仍散在 [content.http.ts](/H:/工作/天异/uniapp+vue新架构/src/implementations/content.http.ts) 和多份文档里。
2. **没有明确的后端阅读顺序**
   - 现在后端读这个文件，仍需要自己推断哪些 DTO 对应哪个 endpoint。
3. **没有显式字段归属总段**
   - 哪些字段是正式后端字段、哪些仍是前端上下文、哪些是 UI preset，目前主要写在文档，不在唯一源文件里。
4. **没有显式“当前不纳入 contract 的字段”段**
   - 例如 `owner / contract / chain / tokenStandard / traits / provenance` 仍不属于正式后端字段，但这个结论没有写进唯一源文件。
5. **没有显式“当前正式分页口径”总段**
   - `page + pageSize` 目前只在请求 DTO 注释和文档里分散出现。

### 3.3 transport 与 service 层还存在的硬编码/分裂

[src/implementations/content.http.ts](/H:/工作/天异/uniapp+vue新架构/src/implementations/content.http.ts) 当前仍直接硬编码：

1. `/api/content/scene`
2. `/api/content/resource`
3. `/api/content/list`
4. `/api/content/action/notice-read`
5. `/api/content/action/service-reminder-consume`

这意味着：

1. 即使 contract 文件成为单一真值源，HTTP adapter 仍会并行保留第二份路径真值。
2. `P11.3` 必须把这些路径收回到 contract 文件导出的 endpoint 常量或等效结构。

### 3.5 当前 handoff 仍让后端面对多套正文

[backend-handoff/README.md](/H:/工作/天异/uniapp+vue新架构/backend-handoff/README.md) 当前要求后端阅读：

3. `后端联调环境与启动说明.md`
4. `后端移交内容清单.md`

这与本轮目标冲突：

1. 后端应只需要面对一个正式 API 源文件。
2. handoff 目录应该只负责环境、启动、校验，不该继续承担正式正文。

### 3.6 真实留出接口仍未完全闭环

[scripts/check-content-http-link.mjs](/H:/工作/天异/uniapp+vue新架构/scripts/check-content-http-link.mjs) 当前已经验证 5 个正式接口，但仍有两个缺口：

1. 它自己的 endpoint 清单没有直接从 contract 文件读取，而是脚本内部再写一份。
2. 它依赖环境变量样本才能完整验证 `resource` 和两个 `action` 接口。

结论：

1. 这个脚本可以继续保留。
2. 但后续应和单一真值源显式对齐，不再形成第二套隐形接口清单。
