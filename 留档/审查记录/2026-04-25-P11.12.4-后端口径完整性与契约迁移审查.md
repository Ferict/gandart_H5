## 4. 当前前端真实 API 状态

当前前端真实 API 落地链仍是内容域 5 个抽象聚合接口：

1. `GET /api/content/scene`
2. `GET /api/content/resource`
3. `GET /api/content/list`
4. `POST /api/content/action/notice-read`
5. `POST /api/content/action/service-reminder-consume`

当前后端契约是 190 个旧 H5 业务 wrapper。

核心差异：

1. 当前前端是聚合接口。
2. 后端契约是业务 wrapper。
3. 二者不能直接一一替换。
4. 必须通过 `backend-http implementation -> adapter normalized model -> runtime/service -> view-model -> UI` 接轨。

## 5. 当前阻塞

在开始真实代码接轨前，有一个硬阻塞：

1. 必须先决定是否用外部最新版覆盖项目内 19 个不一致契约文件。

如果不先处理这个问题，后续 adapter 可能基于旧 contract 写错字段。
