## 当前正式接口范围

- `GET /api/content/scene`
- `GET /api/content/resource`
- `GET /api/content/list`
- `POST /api/content/action/notice-read`
- `POST /api/content/action/service-reminder-consume`

## 建议后端重点先看

### scene

- 首页、活动页、个人页、设置页、服务页首屏聚合。

### resource

- 统一详情与动作详情承载。
- 资源类型是否要求实现详情查询，以 contract 内 `CONTENT_API_RESOURCE_TYPES.detailSupport` 为准。
- `category` 当前只作为引用或筛选目标；`user_profile` 当前后置，不要伪造空 payload 冒充已支持。

### list

- 市场列表、公告列表、个人资产列表统一分页查询。
- 当前正式口径仍是 `page + pageSize`。
- 后端接收的 query 是扁平字段，不接前端内部 `sort.field / dateRange` 嵌套结构。

### action

- 公告已读写回。
- 服务提醒消费写回。
- 两个写回接口都不新增 `userId / accountId`，后端从登录态、token 或 session 解析当前用户。
- 业务失败优先返回 HTTP 2xx + `ContentEnvelope.code != 0`。
